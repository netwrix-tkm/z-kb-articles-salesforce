Symptom

The following error is prompted in any Azure-based MP:

SharePoint Online: Cannot add the item for auditing due to the following error:
AADSTS900382: Confidential Client is not supported in Cross Cloud request.

Causes

The bug that has been fixed in Netwrix Auditor 10.6 build 12299. 
OR
The NationalClouds.xml files are misconfigured for GCC\GCCH MPs.

Resolution

Check you Auditor version. If it is older than 10.6.12299, upgrade to the lates version. 
Adjust the EndpointType value in corresponding NationalClouds.xml files. Refer to the GCC\GCCH article for additional information on templates: Azure and EXOnline: GCC and GCC High.
