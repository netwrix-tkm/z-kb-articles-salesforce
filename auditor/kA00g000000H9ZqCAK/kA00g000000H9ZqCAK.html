You cannot launch the created product task due to the following error: The Task Scheduler Service is not running. (Exception from HRESULT: 0x80041315)
Common causes of this issue are: - The Task Scheduler service is not running; - Created task is corrupted.
In order to resolve the issue please make sure that the Task Scheduler service is running. If it does appear to be running follow the instructions below to re-create the task.
1. Delete the existing product task from the Task Scheduler; 2. Create a new text file and paste to it the following: <?xml version="1.0" encoding="UTF-16"?>
<Task version="1.1" > <RegistrationInfo>
<Author>USERNAME</Author>
</RegistrationInfo>
<Triggers>
<CalendarTrigger>
<StartBoundary>2014-01-17T03:00:00</StartBoundary>
<Enabled>true</Enabled>
<ScheduleByDay>
<DaysInterval>1</DaysInterval>
</ScheduleByDay>
</CalendarTrigger>
</Triggers>
<Principals>
<Principal id="Author">
<UserId>DOMAINUSERNAME</UserId>
<LogonType>InteractiveTokenOrPassword</LogonType>
<RunLevel>HighestAvailable</RunLevel>
</Principal>
</Principals>
<Settings>
<DisallowStartIfOnBatteries>false</DisallowStartIfOnBatteries>
<StopIfGoingOnBatteries>false</StopIfGoingOnBatteries>
<IdleSettings>
<Duration>PT600S</Duration>
<WaitTimeout>PT3600S</WaitTimeout>
<StopOnIdleEnd>false</StopOnIdleEnd>
<RestartOnIdle>false</RestartOnIdle>
</IdleSettings>
<Enabled>true</Enabled>
<Hidden>false</Hidden>
<RunOnlyIfIdle>false</RunOnlyIfIdle>
<WakeToRun>false</WakeToRun>
<ExecutionTimeLimit>PT259200S</ExecutionTimeLimit>
<Priority>5</Priority>
</Settings>
<Actions Context="Author">
<Exec>
<Command>C:Program Files (x86)NetWrixAD Change ReporterADCR.exe</Command>
<Arguments>-dn=DOMAIN.COM</Arguments>
<WorkingDirectory>C:Program Files (x86)NetWrixAD Change Reporter</WorkingDirectory>
</Exec>
</Actions>
</Task> 3. For the following tags please add your specific information: <Author>USERNAME</Author> -specify netwrix service account <UserId>DOMAINUSERNAME</UserId> -specify netwrix service account <Command>C:Program Files (x86)NetWrixAD Change ReporterADCR.exe</Command> - make sure that path to ADCR.exe specified correctly
<Arguments>-dn=DOMAIN.COM</Arguments> -specify target domain name
<WorkingDirectory>C:Program Files (x86)NetWrixAD Change Reporter</WorkingDirectory> - make sure that path to AD Change Reporter folder specified correctly 4. Save this file in .xml format; 5. Import the task from the created xml-file (Open Task Scheduler | right click on the Task Scheduler Library | select xml | Open).
