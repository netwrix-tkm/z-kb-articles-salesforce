General collector overview

Event Log Collector (ELM) is a relatively simple but powerful tool that allows collecting data that other collector cannot. It is not recommended to use ELM instead of other collectors but only to complement them.

Special things to consider

  • ELM does not perform any data analysis. All it does is collecting events as per selected filters and saving them in SQL/LTA

  • ELM does not produce Activity Records. Therefore, ELM data cannot be found using Search in Netwrix Auditor.

  • ELM can send Alerts, but unlike other collectors it does not store the data collected for alerts in SQL/LTA. If you want to receive alerts AND store the data in SQL/LTA for reporting needs, the same filters have to be configured in both Audit archiving filters and Alerts.

Initial product configuration

ELM is not a part of the main Netwrix Auditor UI. The configurator can be found in the Start menu.

Audit Database configuration

  • SQL database for ELM has a predefined name Netwrix_Auditor_EventLog that cannot be changed.

  • Authentication method and User name are taken from the main Netwrix Auditor UI (configuration.xml) and cannot be customized.

  • ELM database retention is independent from the rest of the product.

"Write event descriptions to Audit Database" checkbox makes ELM save the whole event body to the SQL Database. Good thing is that reports will display more data; bad thing is that the database will grow much faster. This option should only be enabled when absolutely necessary.

​​​​​​​Network traffic compression

The compression service is installed on the target machine in the C:\Windows\NWAgentShare folder. The service name is Netwrix Auditor Event Log Compression Service (nwagentsvc). The default state is "Stopped". Collected events are saved to the same folder before transferring to the host.

Troubleshooting

If you want to see processed events in the trace log file, set TraceLevel 100 (dec).

All configured filters can be found in the %WORKING FOLDER%\Event Log Management\Rules folder. If you need to check what your customer configured, request the above folder and put it into a similar folder in your lab, so you could see the filters in the product UI, which is more convenient than reading XML.
