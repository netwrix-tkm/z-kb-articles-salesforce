Symptom and Cause

Inactive Users Tracker uses LastLogonTime AD attribute to decide when was the last time user was active. Certain actions (such as Logon to Office 365 on hybrid Environments) do not affect this attribute, so IUT perceives these accounts to be inactive and locks them.

Solution

This issue can be worked around by omitting either the user or the OU from the monitoring scope of Inactive User Tracker. This will prevent IUT from acting on account in any way, making it impossible to set the locked status for specified accounts. For additional information on omit lists for Inactive Users Tracker, refer to the following article: Exclude Objects from Monitoring Scope — Inactive Users.
