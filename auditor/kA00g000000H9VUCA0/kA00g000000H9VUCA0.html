Question

How to use wildcards in Netwrix Auditor reports? What are the wildcards?

Answer

Netwrix Auditor reports are based on SQL Server Reporting Services — the same wildcards apply as the wildcards used in SQL queries.

Wildcard

Function

Matched results

%

The % wildcard matches all characters beginning at that position and/or bounded by the next character. See results for the A%E query.

  • AE

  • ABE

  • ACE

  • ABCDE

_

The _ underscore wildcard replaces any single character at that position. See results for the A_E query.

  • A1E

  • ABE

  • AAE

  • AOE

[***]

The [***] square brackets wildcard will match a single stated character at the position in the string. See results for the A[ABC]E query.

  • ABE

  • ACE

  • AAE

  • Not ADE, AEE, etc.

[*-*]

The [*-*] square brackets wildcard will match a range of characters at that position in the string. See results for the A[A-D]E query.

  • ABE

  • ACE

  • AAE

  • ADE

  • Not AEE, AOE, etc.

[^*-*]

The [^*-*] square brackets wildcard will match any character outside of the specified range. See results for the A[^A-D]E query.

  • ARE

  • AGE

  • AOE

  • Not ABE, ACE, AAE, ADE.
