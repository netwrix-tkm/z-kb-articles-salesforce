The customer wants to add his logo to the two PDF reports generated by his GB appliance (Maven in this case).

Those reports are named Greenbone Security Report (GSR PDF) and Greenbone Executive Report (GXR PDF) and can be generated using the GB web console:

To replace the logo on the reports we will need to access the Maven appliance via SSH, using Putty. We’ll also need to use WinSCP to transfer the files.

Follow the steps described below to replace place your logo on the report.

1) Access the Greenbone OS Administration console and Enable SSH access.

a.       On your hypervisor, access your MAVEN console, go to Setup/Services/SSH and enable SSH State (you’ll be asked to set a now Admin password)

2) Enable Superuser account: Now go to Advanced/Support/Superuser and enable Superuser State (you’ll probably be asked to set a password again)

3) Using Putty, connect to your MAVEN appliance. See the picture below for more details about how to configure the connection.

a.       Log in using the admin credentials (admin / admin by default)

4) Once logged in, go to Advanced / Support / Shell. Click Continue on the next screen.

5) Use now the command ‘su’ to access the privileged mode. Use the Superuser account password you set on step 2.

a.       Select Continue on the next screen.

6) You are now on privileged mode. Next step will be modifying the attributes of a couple of directories containing the files on the GSR PDF and GXR PDF reports. Follow steps below in order to get that done.

a.       Using the console, go to /usr/share/gvm/gvmd/report_formats/ and list its subfolders.

We need to modify the attributes for those 2 directories outlined in blue, so we’ll use the command chmod to modify them. (i.e. chmod 777 ebbc7f34-8ae5-11e1-b07b-001f29eadec8/)

b.       Once this is done, we can list again to check that in fact attributes have been modified.

Now we are ready to connect using the WinSCP.

7) Open the WinSCP, and configure a new connection using your server’s credentials.

a.       You will need to go to Advanced Site Settings / Shell and set /bin/sh or you’ll blocked when trying to connect.

8) Now that you are logged in, go to the folder where all the report files are located: /usr/share/gvm/gvmd/report_formats/

a.       Access the folder with the GSR Report: 35ba7077-dc85-42ef-87c9-b0eda7e903b6 and remove the Greenbone default logo files:

·   page_header_logo.png

·   title_page_logo.png

b.       Now copy the two .png files with your logo from your computer to that same folder. Please check below the details about those two .png files with your logo.

·title_page_logo.png
·~ 129 x 177 pixels

·page_header_logo.png
·~ 200 x 215 pixels

c.       Access now the folder with the GXR Report: ebbc7f34-8ae5-11e1-b07b-001f29eadec8 and remove the Greenbone default logo files:

·   page_header_logo.png

·   title_page_logo.png

d.       Now copy the two .png files with your logo from your computer to that same folder.

9) Using your web browser, access your Greenbone security manager, log in, and go to Scans / Reports. Select your report and hit the Download filtered report button. Now select the Report Format (GSR PDF or GXR PDF), hit OK, and your PDF report will be generated WITH YOUR LOGO.
