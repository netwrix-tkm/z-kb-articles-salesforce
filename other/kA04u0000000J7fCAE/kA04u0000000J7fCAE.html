Issue: Summary:
This is a fix for the SQL DC Customer queries not correctly handling column names that have spaces.
 
Behavior Before Fix:
Custom query of Selecting from column [column name] would collect data from [column_name]
 
Behavior After Fix:
Custom query of columns named with spaces are collected correctly.  Note that the results will show a "_" in the column name instead of a space, but the query is targeting the correct column.

Issue:
Core automatically replaces spaces and other special characters with a "_".  We now handle spaces, but there will still be problem if there are two columns being selected like [column_name] and [column name].  The 8.2 SQL DC customer query wizard has additional functionality to prevent conflicts that the 8.1 code does not.
For reference, the Supported characters are (everything else is replaced by _):
a-z A-Z 0-9 ! @ $ % ^ & * ( ) " / ? < > | _ - = + { }
