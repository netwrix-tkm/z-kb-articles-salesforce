Issue: Summary: SI FS Activity for SA Policy Issue - SI Resolves FQDN to Shortname
Behavior before Fix:
Previously when creating 'File System StealthAUDIT' policy SI Agent will create section 'FILE_MONITOR_SA' in the SBTFileMon.ini file and will fill 'HOST' parameter inside this section with FQDN name of the monitored host.
Behavior after Fix:
Now it can set FQDN or NetBIOS name of the host depending from the 'File System StealthAUDIT' Policy settings. This setting can be configured on the 'File System Agents' tab.
Changes since Last Hotfix:
I-4670 NetBIOS / FQDN names settings in the FS Audit policy are exported not correctly - all agents are marked as like they have FQDN name selected
