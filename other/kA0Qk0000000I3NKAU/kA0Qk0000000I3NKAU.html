Symptom

After enabling Compromised Password Checker, the following warning is prompted in the Netwrix Password Policy Enforcer event log:

Password verification will be performed on the domain controller %DC_FQDN%.

Causes

  • The Password Policy Enforcer Mailer service is not installed.

  • The fully qualified domain name (FQDN) of the domain controller the Password Policy Enforcer Mailer service is installed on is incorrect.

Resolutions

  • PPE Mailer is required for Compromised Password Checker to operate. In case it was not installed previously, refer to the following article for additional information on installation: Administration − Mailer ⸱ v10.2.

  • Review the FQDN of the domain controller PPE Mailer was installed to:

    1. In the main Password Policy Server window, click Compromised Password Checker.

    2. In the General tab, review the Domain Controller field − make sure the (FQDN) of the domain controller the Password Policy Enforcer Mailer service is installed on is correct.
