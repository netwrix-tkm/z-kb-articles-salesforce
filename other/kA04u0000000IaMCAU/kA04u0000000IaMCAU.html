Issue: Summary: SI agent using huge amounts of memory
Behavior before Fix:
In old implementation agent was trying to parse the whole tsv file at once and then processing collected entries. As the result memory consumption could heavily increase for handling tsv files from NAS devices.
Behavior after Fix:
SI agent accumulates entries for single event then process it and then repeat. As the result memory consumption should be significantly decreased.
