Summary: How to configure multiple SAM clients to collect activity for a single NetApp

Issue: Customer wants to load balance activity events from a large NetApp device or create a fail over system to ensure all events are delivered.  

Instructions:

Step 1:
Install SAM on two separate Windows servers and configure them to both monitor the same NetApp filer/cluster

Step 2: 

Load Balancing
The IP address of the secondary SAM Agent needs to be added to the primary servers parameter of an external engine.
 
if multiple SAM agents are defined as primary servers, they will receive events in a round robin fashion.
If one fails, the other(s) will continue to receive events and no events will be lost. This would also help balance performance for a busy filer.

Fail Over
The IP address of the secondary SAM Agent needs to be added to the secondary servers parameter of an external engine.
 

Step 3:
To collect from both SAM servers, StealthAUDIT should have two separate FSAC scans targeting the same NetApp.  
Adjust one scan's proxy setting to the primary SAM server, and the other scan's to the secondary.
