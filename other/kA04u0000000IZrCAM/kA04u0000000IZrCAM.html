Issue: Summary: Ignore failed logins for unresolved user names not working for Brute Force
Behavior before Fix:
The error was inside the logic responsible for new attack detection. The code which responsible for analyzing individual event ignores unresolved account as it must but when attack recognizer calculates the number of failed logins to understand if this is enough for reporting new attack it goes through all events collected by corresponding policy and ignores "Ignore failed logins for unresolved user names" option.
Behavior after Fix:
Now code modified in the way to take this option into account when getting list of collected logins for new Brute Force attack recognition.
