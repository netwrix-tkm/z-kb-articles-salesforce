Issue:

When using the Active Directory Migration Tool for Exchange users were not being marked as deleted when migrated to a new domain.  This new setting will track SID history and mark users as deleted from the originating domain. 

 

Instructions:

  1. Unblock the hotfix zip file in the windows property dialog if an unblock button exists there.
  2. Close all instances of StealthAUDIT (check task manager under processes for all users)
  3. Exit StealthAUDIT console
  4. Copy and replace files from the folders in this hotfix to the corresponding folders in SA Install dir.
  5. Check new option in ADI data collector wizard "Support AD Migration of user and group objects based on SID history data"
