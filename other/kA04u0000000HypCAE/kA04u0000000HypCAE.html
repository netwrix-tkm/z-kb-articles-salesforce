Summary: Fixes issue found where Cisco credentials would be exposed in SbPAM logs in certain scenarios.

Affected Versions: SbPAM 3.0.9118 and earlier

KB Type: Cumulative Update

Submitted by: Dan Piazza

Download Link: https://downloads.stealthbits.com/access/files/SbPAMHotfixes/3.0/SbPAM.3.0.9120.zip

Work Items/Escalations:

  • 6887 SbPAM - Credentials exposed in logs
  • 6778 SbPAM - Linux Activity Token and Managed Login Account types use a username instead of full loginAccountFormat for SSH log-in

Instructions:

Download Zip file to SbPAM Server.

NOTE: If running HA configuration, please refer to HA instructions.

  1. Extract all files.
  2. Run Sb_Setup.exe as Administrator.
  3. If upgrading from 3.0sp2 then after setup completes, login to SbPAM Web UI to allow services to reregister.

Configuration instructions for 2824:

2824 Allow authenticators to be used that aren't assigned

To change the authenticator behavior to pre-sp3 behavior (any user can authenticate with an external authenticator), update the Web C:\Program Files\Stealthbits\PAM\Web\appsettings.json file.

NOTE: This path may vary if you installed to a non-default location.

Add the following line before the closing } , "StrictExternalAuth": "False" For example if your existing appsettings file looks like this: 

{
   "StrictExternalAuth": "False"
}

NOTE: Make sure to add a comma if there are other properties.

BEFORE: 

{
   "Example": "value"
}

AFTER: 

{
   "Example": "value",
   "StrictExternalAuth": "False"
}
