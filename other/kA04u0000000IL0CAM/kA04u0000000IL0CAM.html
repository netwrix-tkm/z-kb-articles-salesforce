Summary:

LDAP Monitor: Perpetrator Filter Handling from SAM

LDAPM policy. Perpetrator filter by context does not work.

Role based access enhancement: Prevent Console operator from purging Alerts

Behavior before Fix:

SI-4158and I-6653: Agent use perpetrators without SID to compose invalid rules for AD Monitor

SI-4171: non Admin accounts can clear alerts

Behavior after Fix:

SI-4158 and I-6653: Agent stop producing invalid rules from perpetrators without SID and evaluate such filters by DN

SI-4171: only Console Admin can clear Alerts list

Instructions:

The following SI modules should be patched or upgraded:

 

* Apply patch for SI Enterprise Manager

* Apply patch for SI Windows Console

* Upgrade SI Windows Agent

 

Applying patch for Enterprise Manager:

  1. Stop 'SIEnterpriseManager' service.
  2. Extract files from attached 'SIEnterpriseManager' folder to the SI Enterprise Manager folder
  3. Start 'SIEnterpriseManager' service

 

Applying patch for SI Windows Console:

  1. Close the SI Win Console
  2. Extract files from attached 'SIWinConsole' folder to the SI Win Console folder
  3. Start SI Win Console.

 

Applying patch for SI Windows Agent:

7.0.1 – Upgrade SI Agent to 7.0.1.662 or newer
