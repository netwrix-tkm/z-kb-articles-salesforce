Issue: Summary:
Summary: Fixed a bug in connection profiles where using FQDNs did not work if there was more than one credential in the profile.
Issue/Request: Domain logic was ignored when multiple domains were targeted, and only the NETBIOS name was allowed.
Fix/Enhancement: We now reorder credential sets in the background as follows: when an FQDN is available and matches a credential at the host, it comes first; then a credential matching the NETBIOS name; then other, nonmatching credentials.
