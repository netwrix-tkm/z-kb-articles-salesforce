Summary:  Error message about parsing datetime on import

Before Fix: When scanning on a proxy/remote host that has a time setting to be displayed with periods instead of colons, FSAA stores the ScanTime value incorrectly and prevents bulk import from working correctly.

After fix: FSAA should store the datetime correctly, regardless of the computer settings on the execution host.

Comments:

THIS HOTFIX HAS A PREREQUISITE OF HOTFIX SA_9.0_018. PLEASE APPLY THAT HOTFIX PRIOR TO APPLYING THIS ONE. SEE KB FOR THAT HOTFIX FOR MORE DETAILS

This version of FSAA includes a change that also requires a change to SDD add-on. Please upgrade your SDD add-on to version 9.0.0.33 or later. FSAA will be compatible with older versions of SDD but you will see a warning message that says the following:
“Cannot find interface for temporary file scan, SDD Components may be out of date. Continuing scan with old interface...”

Instructions:

  1. Unblock the hotfix zip file in the windows property dialog, if an unblock button exists there. 
  2. Close all instances of StealthAUDIT (check task manager under processes for all users)
  3. Update all DC files in your %SAInstallDir%DC folder
  4. Update all PrivateAssemblies in your %SAInstallDir%PrivateAssemblies folder.
  5. Run the ‘Update Proxy Service’ query under the Maintenance tab in the FSAA data collector against your proxy servers (if running proxy service).
    1. PLEASE NOTE THAT THIS HOTFIX INCLUDES A CHANGE THAT MAY REQUIRE YOU TO RUN THE UPDATE PROXY SERVICE TWICE TO UPDATE SDD COMPONENTS ON THE PROXY SERVER. YOU SHOULD RECEIVE A WARNING ON THE FIRST RUN INDICATING THIS.
  6. Delete tier 2s on proxy/remote host (if applicable). You do NOT have to delete the tier 2 on the local SA Console.
  7. Re scan affected hosts.
