Instructions: 1. Unblock the hotfix zip file in the windows property dialog, if an unblock button exists there.
2. Close all instances of StealthAUDIT (check task manager under processes for all users)
3. Exit StealthAUDIT console
4. If the installed version of SDD is lower than SDD 8.1.0.34 - Uninstall SDD add on from SA Console
5. Navigate to Sensitive Data Folder in SA Install Dir > there may be leftover files that need to be cleaned up prior to installation of new sdd package.
6. Copy and replace files from the DC folder in this hotfix, to the DC Folder in SA Install Dir
7. Copy and replace files from the PrivateAssemblies folder in this hotfix, to the PrivateAssemblies folder in SA Install dir.
8. Install included SDD package on the SA Console(x86)
9. Take a backup of the files in SPAA directory on Target SharePoint server (in case the customer has config files that have been customized)
10. Uninstall SPAA Agent and SDD Add- on from target SharePoint Server(s), delete any leftover files that are not the license file.
11. Install and configure SPAA Agent from this package
12. Install included SDD Add package on target SharePoint Server (x64)
