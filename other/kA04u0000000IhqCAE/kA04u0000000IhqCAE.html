Summary: New Password checking PS API not functional

Issue: Summary:
New Password checking PS API not functional
Behavior before Fix:
There is a logic in LDAP monitoring that if we see that event is initiated by the local system account then we ignore this event and just call original function and return. The problem was that code which is responsible for the functionality to handle 'special' LDAP requests to evaluate EPE rules was executed AFTER local system account validation and as a result was unreachable when LDAP monitoring is enabled.
Behavior after Fix:
Now logic in this place changed to handle such situation properly.
