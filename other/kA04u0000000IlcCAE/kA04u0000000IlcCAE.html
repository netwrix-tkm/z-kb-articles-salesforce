Summary: SI-Alerts SBTService is "Stopped"

Issue: Summary:
SI-Alerts SBTService is "Stopped"
Behavior before Fix:
When ini file has two sections: first SI with office filtering and the second SA without office filtering next situation can happen. When permission event arrives security difference is calculated. Event is passed to office filtering that might trigger several events. When events are extracted from office filtering security difference is calculated one more time. This might lead to the state when the difference is null. Next goest SA section. The same permission event is passed to SA section and as it doesn't have office filtering event is directly sent to the output but with incorrect security difference. In older versions this lead to crash due to AV in latest versions event will be skipped or incorrectly written to file.
Behavior after Fix:
Now events should be processed correctly without SBTService crash
