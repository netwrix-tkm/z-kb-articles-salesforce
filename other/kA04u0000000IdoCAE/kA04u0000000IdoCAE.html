Issue: Title:  SIWindowsAgent_5.0.0.789_5.1.0.414_SI-3536
Summary:
SI file activity not resolving perpetrator SID
Behavior before Fix:
Problem description:
To search account by SID when account is from the trusted Domain it is necessary to set ReferralChasing property of DirectorySearcher object to 'ReferralChasingOption. All' value. But for FS monitoring it wasn't done.
Behavior after Fix:
Code modified to set this option in case when SID is not from the current domain.
