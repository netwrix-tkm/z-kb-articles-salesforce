Summary: AIC - Hide Email Options from Owners When Email is not Configured; AIC - Disable Links for Resource Owners

Issue: Summary:

  • Ticket: AIC - Hide Email Options from Owners When Email is not Configured
    • Issue: If there is no email server configured in the AIC we still show the option for owners to subscribe to email alerts ("Send me alerts for this resource once per week"). This should only be shown if it will work, which means email is configured.
    • Fix: If SMTP Server field is empty in the AIC Notifications page, the alerts option won't show in the Owner's Update dialog.
  • Ticket: AIC - Disable Links for Resource Owners
    • Issue: In certain cases resource owners do not need to be able to drill into a resource because the information is overwhelming and confusing. For non-technical owners we should be able to disable these links.
    • Fix: A new parameter was added in the Web.config file (OwnershipAuditEnabled), it controls whether or not the resource links are enabled (true, false). The parameter is set to true by default. The areas affected are in the Ownership portal: The owned resource main grid, and the bottom tabs (Pending Requests, Request History, and Review History).
