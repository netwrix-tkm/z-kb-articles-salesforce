Summary: How to clear & collect StealthAUDIT's core logs.

Issue: for certain issues, such as reporting, notification issues, core hangs, etc., you may need to gather core logs from StealthAUDIT.  

Instructions: First, clear the core logs, if possible.  This makes it easier for development to analyze.  

  1. Ensure StealthAUDIT is in debug mode (Settings > Other Setttings)
  2. End all instances of StealthAUDIT (check task manager for scheduled instances or interactive instances opened by other users)
  3. Navigate to the SADatabase folder in the StealthAUDIT installation directory (%sainstalldir%SADatabase)
  4. Delete the Logs folder
  5. Reproduce the issue you want to gather core logs for
  6. gather logs from the Logs folder in SADabase (%sainstalldir%SADatabase\Logs)
