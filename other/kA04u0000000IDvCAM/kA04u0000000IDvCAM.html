Issue:
When running the AD_SensitiveSecurityGroups job in the customers environment they receive the following error for the Calculate Effective Membership analysis task.
Subquery returned more than 1 value. This is not permitted when the subquery follows =,!=,,>= or when the subquery is used as an expression.
After running the query in SQL, we were able to determine the error is referring to line 27 of the script
SET @DNSAdminSID = (SELECT ObjectSID FROM [dbo].[SA_ADInventory_GroupsView] WHERE [SamAccountName] LIKE '%DNS Admins')
This subscript is producing more than one result because they have multiple DNS Admins appearing (multiple domain environment). The current query will need to be altered to take this into account.
 

Instructions: 1. Exit StealthAUDIT
2. Copy and replace file in this hotfix to the corresponding folder
     folder paths have been included with this HF. The path that starts with "jobs" needs to be replaced.  The "Instant Solutions" will need to be replaced if the customer deletes the SS and recreates it. 
3. Restart StealthAUDIT
