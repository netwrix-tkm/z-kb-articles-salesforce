Summary: Added support for TLS v1.2; job connection and report generation errors.
Issue/Request: Currently, StealthAUDIT does not support using TLS v1.2 for communicating with a database server. As this standard is set to replace earlier versions of TLS in the coming years, customers are starting to transition to 1.2. Additionally, Turkish language environments were not honoring SQL credentials or generating reports.
Fix/Enhancement: We are adding compatibility with the TLS 1.2 standard to SA v8.0 and all subsequent versions. Turkish-language environments can connect to SQL with correct credentials and generate reports.
