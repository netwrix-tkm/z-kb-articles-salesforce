Summary: AIC not showing activity from an EMC device?

Issue: AIC isn't showing activity from an EMC device.  

The issue occurs when we audit activity on a Celerra host with the "Virtual Data Mover" feature enabled.  

When that is the case, in front of all of our activity log entries we see something like the following:

C:\root_vdm_1\folder1\folder2\folder3

However, when FSAA collects permissions and the folder structure for the device, it saves the path without the VDM name, as such:
C:\folder1\folder2\folder3

So, when the AIC tries to load up the activity corresponding to a particular path, it doesn't find any.  
 

Instructions: To resolve the issue,

1) open the SBTFileMon.ini file on the machine where the file monitor is installed.  
2) add a line to the section for the appropriate host where you can specificy the VDM Names to be removed from the path


the syntax is: 

VDM_NAMES=root_vdm1,root_vdm2


WARNING! If a folder exists in the root of C: that has the same name as the VDM, its activity will now have an incorrect path.
