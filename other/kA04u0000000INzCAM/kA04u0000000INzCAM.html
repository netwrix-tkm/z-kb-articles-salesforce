Issue: Summary: Have all SI Analytics details in Splunk
Behavior before Fix:
Information sent to SPLUNK from SI is just the high-level information about the threat detected.
Behavior after Fix:
Now it is possible to get in SIEM message for Analytics list of events which triggered new attack. It can contain one event for analytics like Golden Ticket or several events for analytics like Brute Force Attacks. This list is only available for newly detected attacks!
List of events is provided in the form of xml string using the following format:

For Authentication based analytics it includes the following fields:
[isSuccessful]
[loginType]
[fromHost]
[fromHostProtocol]
[fromHostIp]
[toHost]
[toHostIp]
[accountSid]
[accountName]
[utcTimeLogged]
[treeOrDomain]
[originatingServer]
[status]
[eventsCount]
[isUserExist]
[operationResult]
[agentTimeLogged]
For File System based analytics it includes the following fields:
[utcTimeLogged]
[treeOrDomain]
[fsHost]
[fsHostIp]
[fsObjectType]
[fsPath]
[fsPathNew]
[fsOperation]
[accountSid]
[accountName]
[isSuccessful]
[processName]
[fromHost]
[fromHostIp]
[fromHostProtocol]
[agentTimeLogged]
To get this list of events in the SIEM message it is necessary to use custom SIEM template with %ANALYTICS_EVENTS% replacement token inside.
For example, current Splunk templates (both "AuthAnalytics - V2.0 for Splunk SIEM.txt" and "FileAnalytics - V2.0 for Splunk SIEM.txt") can be modified from value like this:
%SYSLOG_DATE% %HOST% %PRODUCT% - SI Analytics_%ALERT_NAME% ... AlertText=%ALERT_TEXT%
to this:
%SYSLOG_DATE% %HOST% %PRODUCT% - SI Analytics_%ALERT_NAME% ... AlertText=%ALERT_TEXT% Events=%ANALYTICS_EVENTS%.
