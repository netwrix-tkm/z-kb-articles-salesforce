Greenbone Trial Form Usage

NOTE: The Greenbone trial form should only be completed if the prospect is evaluating a physical device or an EXA virtual device or larger. The evaluation of a Maven or CENO does not require the trial form to be completed.

To be completed in the trial form:

  • Period of Trial:
  • Check Virtual Appliance
  • Check The customer confirms to delete within 30 days...
  • Signed

NOTE:Period of the trial always 2 weeks, active from the time they gen the license

The POC Manager will check if NNT has a Greenbone image available for the POC, i.e. we have MAVEN and CENO images but no DECA. If the correct image is not available, then the POC manager will request an image in an email to Lena Anders and Carrine with the account manager copied.

Information to be included in the email:

lena.anders@greenbone.net
carinne.louth@greenbone.net

Email Subject Line: NNT & Greenbone Eval Licenses -

If required, the Greenbone trail form should be attached.

If required, a request for a downloadable image.

Greenbone will respond to this email with a trail key for the prospect that NNT will build into the relevant Greenbone image.


Support tells the prospect that their POC system is being built, provides details of recommended specs and asks the prospect to register for the Members Area

Create a Group for the Prospect

In preparation for the download, support creates a Members Area group for the trial

  • Users > groups > Search for Reminger
  • Click the link to enter theReminger group
  • Press the Save and copy button
  • Search for the copied group, the group will be namedReminger (2)
  • Click the nameReminger (2) and rename the group to the name of the prospect

Create the Prospect's Image

Support updates a generic Vulnerability Tracker image of the type required with the key provided from Carrine/Lena

  • Name the image to include the name of the POCing organisation - exampleGSM-MAVEN-5.0.7-iTrustech.ova
  • See the details of how to upload a key via this confluence page -Support Process - Vulnerability Tracker Subscription Key Export/Import

Upload the Prospect's Image to the Members Area

Support uploads the created trial image to the Members Areas

  • Goto DOCman > Files and upload the prospect's image to Joomla
  • There is a Vulnerability Tracker folder off the root folder, upload to this location.

Creates the Download for the Prospect

  • Goto DOCman > Documents and move to:
  • Software > Vulnerability Tracker > Software Installers > Server Installers > Linux > Maven
  • Check a box next to an existing trial download and click the duplicate button at the top of the page. In the popup box type VT MAVEN
  • A duplicate is now visible, click the link of the duplicate to enter the properties
  • In the 'file' section, click the 'Select existing file' link and select the trail download that was created and uploaded.
  • In the 'Permissions' section change the 'Access' to Groups
  • Change the 'This document can be viewed by' to the group created for the trial

Edit Prospect's User Account for Download

Once the prospect has registered for the Members Area, alter their permissions for the user from Community Builder: User Management

  • Search for the prospect's registered account and edit their account
  • In the groups section, hold down the control key (very important) and highlight 'VT Download' and the group name created for the prospect in theCreate a Group for the Prospect stage
    Save and close

The prospect should now be able to see their download in the Members Area at the following path:

Vulnerability Tracker > Software Installers > Server Installers > Linux > Maven
