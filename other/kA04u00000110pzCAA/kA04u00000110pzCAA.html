Formerly SecureONE

When reaching out to systems for any action, Netwrix Privilege Secure for Discovery will use the FQDN (fully-qualified domain name) or the last known IP address. The CN (common name, short name) is never used.

SecureONE scanner and worker services will:

  • Attempt to ping (IPv4), then connect using the FQDN first, if that fails. Timeout for ping, using fping, is 500ms

  • Attempt to ping, then connect using the last known IP address

  • Both FQDN or the last known IP address will be retried once

  • If both ping retries fail, the host is deemed offline and Host Offline error occurs
