Formerly SecureONE

Symptom

When an update to an AD group is made the Update Sequence Number (USN) is incremented. Netwrix Privilege Secure for Discovery utilizes this to identify the change and flattens the updated group to determine suitable access to a given machine.

When a user is moved from one OU to another, an issue can occur that a user loses access to machines that previously they had access to.

Cause

The distinguished name for the user has changed, due to OU move, however, the groups associated with the user has not updated. As these groups have not been updated, the old distinguished name has not been updated either.

Resolution

Make a change to the group that the user should be receiving access from. This can be simply, the addition and removal of a different account. This will trigger Netwrix Privilege Secure for Discovery to recognize an update and therefore perform a re-flattening of the group, restoring access.
