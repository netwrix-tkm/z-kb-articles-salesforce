Cumulative List of fixes included since Service Pack:
- SAFS-992:
1. Added transactions and check pointing into the permissionsComparison analysis
2. Added ability to use #SA_JOB_HOSTS to filter down analysis to only hosts inside the job list
3. Added the ability modify a SQLScript parameter in analysis #2 which tells the job to only analyze resources with changed permissions from parent. By default this is enabled, but can be set back to 0 to get legacy behavior (Analyze all resources regardless of permission changes between parent and child)
4. If running enterprise SQL server, will compress certain large tables with page compression.
5. Filtered resources with DeletedUSN.

  1. Replace the DataAnalysisTask.XML file in the job under GROUP_FileSystem\GROUP_3.Broken Inheritance\JOB_FS_BrokenInheritance with the one provided in the hotfix.
  2. Optional:
    1. To get results only on folders with changed permissions from parent, leave analysis as it is (runs faster).

    3. To scope the results to specific hosts, target the desired hosts (from the list of scanned hosts) in the job's Hosts section.
  3. Rerun the job.
