Issue: Title:   SIWindowsAgent_5.0.0.749_SI-3480
Summary:  LDAP monitoring and nacn_ip_tcp protocol activity
Before fix:
If some RPC call from workstation causing DC to execute LDAP query locally we think that this request comes from workstation. But in fact it is generated internally by LSA to respond to this request.
To capture LDAP Searches we are using hooks for the following functions:
- LDAP_CONN::SearchRequest
- DirSearch
SI Event is reported from DirSearch even if there was no call to SearchRequest. Therefore internal LSASS DirSearch function call was interpreted as client initiated event.
After fix: Now SI Event will be generated only if both hooks are triggered for event.
