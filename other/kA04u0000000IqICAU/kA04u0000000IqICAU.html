Summary: The Following KB Article Explains the difference between NetApp SVM name vs CIFS vs Cluster Node Name

Issue: Multiple network names used with NetApp shares how to determine what name will work best

Instructions: So first thing you will want to do is identify all Node Members in cluster you will need this info when configuring activity agent so you can auto enable Fpolicy this is important to configure since at times Fpolicy will turns itself off!
CLUSTER NODES -------------------------------------------
The following example displays information about all nodes in the cluster:

cluster1::> cluster show

Next we need to know the CIFS server name being used since typically this will be the name registered in DNS with windows this will also supply us with SVM (SERVER VIRTUAL MACHINE) Name

Once you have gathered SVM, CIFS and CLUSTER NODE NAMES you will want to determine what name is being used serve File Data
 
First step is to ping CIFS Name and see if it responds and from what IP next Ping SVM Name and see if this now responds and it IP verify that returned IPs are the same as the SVM DATA LIF
