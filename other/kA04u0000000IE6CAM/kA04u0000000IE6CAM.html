Summary: SI Admin console won't open after an upgrade

 

Issue:  In the EM startup log:  Incoming events queue wasn't created
In the enterprise manager log:  EM is not in running state, refusing incoming events


Instructions:  
In SQL Management Studio:
SELECT * FROM [NVMonitorConfig].[dbo].[EventConsumer]
Check [config] column for reference of 'MySQL'
 
If this Exists Follow these steps:

  1.  Stop Enterprise Manager
  2. Find [PolicyEventConsumer] value for row where 'MySQL' is referenced
  3.  Delete row from [NVMonitorConfig].[dbo].[EventConsumer] where  'MySQL' is referenced

  1. Start the Enterprise Manager this will create a MSSQL row in [NVMonitorConfig].[dbo].[EventConsumer]
  2. Find [PolicyEventConsumer] value for MSSQL row
  3. Stop the Enterprise Manager
  4. Replace the old [PolicyEventConsumer] in [NVMonitorConfig].[dbo].[EventConsumer] with the new MSSQL value

  1.  Start the Enterprise Manager

Note:  collections with type 11 and 9 which are not supported by StealthINTERCEPT should be resolved in 5.1
