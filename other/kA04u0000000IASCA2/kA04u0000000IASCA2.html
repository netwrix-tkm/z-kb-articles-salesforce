Issue: Ticket: AIC Group Membership Changes
Issue: A folder owner with no console access can come into the AIC, navigate to the resource audit for a folder they own, click a group that is managed by the AIC, and change membership on it using change modeling.
Fix: The behavior was changed to:
* An owner that has Administrator or Security Team role will be able to change access to owned resources in Resource Audit/Group Explorer.
* If the owner has Read role or no role assigned, then he won't be able to make the changes through resource audit (the Commit button won't be available in Group Explorer). The owner can still use the Change Access button in the owner's portal.
