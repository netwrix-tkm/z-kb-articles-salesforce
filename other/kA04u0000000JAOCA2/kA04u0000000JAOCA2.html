Issue: DEFEND - update the integration job to use the proper JWT settings
Before we used NancyFX and it didn't want the bearer keyword in the authorization header
Now we use ASP.NET core and it wants it
 

Instructions:

  1. Unblock the hotfix zip file in the windows property dialog, if an unblock button exists there. 
  2. Close all instances of StealthAUDIT (check task manager under processes for all users)
  3. Update all Action files in your %SAInstallDir%Actions folder
  4. Update Instant Jobs in your %SAInstallDir%InstantJobs folder.
  5. Launch StealthAUDIT.
  6. Recreate StealthDEFEND job so it pulls the new files from instant jobs
