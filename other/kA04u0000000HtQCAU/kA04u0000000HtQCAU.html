Summary:

SQL performance issues with XEvents enabled, After XEvent sessions disabled we cannot re-enable from SAM

Behavior Before Fix:

SAM required ALTER ANY EVNT SESSION permission for SQL Monitoring.

Behavior After Fix:

ALTER ANY EVENT SESSION sql permission is now optional. Without permission event session will need to be created manually.  SAM logs a query in sql monitoring log to create session.
