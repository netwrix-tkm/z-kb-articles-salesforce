In this particular case we’ll be converting a CENO image provided by GB as an OVA to a VHD fixed-size image that can be imported into Microsoft Azure. We’ll be enabling SSH access and Superuser credentials previous to the conversion.

Please follow the steps below to get it done:

  • Download the OVA provided by GB.

  • Import the downloaded OVA into your VMware ESXi environment.

 

  • Power On the VM and log in using admin/admin. Hit No when asked.

  • Go to Setup > Services > SSH and enable SSH. Set a new password when asked. This will be your new password to access the Administration console so don’t forget it. Hit Save.

  • Let’s set the superuser credential now. Go all the way back to the main menu, and go to Advanced > Support > Superuser. Enable it and set the password. Just hit continue when prompted. Hit Save.

  • At this point we can turn off the VM. Please make sure to do it at Main Menu > Maintenance > Power > Shutdown.

  • With the VM powered off, hit in the Actions option, and select Export. This will generate 2 files and OVF and a VMDK. Once the export has finished, you should see both files in the Downloads folder on the File Explorer. We’ll need to convert the VMDK one.

  • To convert the VMDK file to VHD (fixed-size) we’ll be using StarWind V2V Converter. Please download it and install it.

 

  • Open StarWind V2V Converter and select Local File option. Hit Next.

  • Hit the Browse (…) button, select the exported VMDK file and hit Open. Now hit Next.

  • Select local file option again and hit Next.

  • No select VHD/VHDX option and hit Next.

  • In the next screen please select VHD pre-allocated image. It’s essential to select this format since it’s the only one compatible with Microsoft Azure. Hit Next.

  • At this point you’ll just need to select the location for the converted file. Please make sure you have enough space. This conversion should generate a 32 GB file. Now just hit Convert and the VHD (fixed-size) file will be generated.

  • If you’re planning to send over the file I would recommend compressing it. It shouldn’t take more than 2 GB when compressed.

  • When creating the instance in Azure, it’s important you create the machine as a Generation 2. (EFI/UEFI) Otherwise you won’t be able to loaded because Generation 1 only supports BIOS.
