Summary: The purpose of this kb article is to provide instructions on producing/collecting memory dumps for different components of the StealthINTERCEPT product

Issue: 
A process memory dump is a snapshot of process memory, which might include additional information such as stack trace, threads, handles etc. It can help software developers to diagnose, identify and resolve problems that lead to application failure.
Typical types of application failures include:
1. Application hangs
2. Application crashes
3. Application consumes more memory than expected or doesn't release memory.
4. Application works incorrectly
Process memory dumps can help to diagnose / troubleshoot cases 1-3. Dump files generally are not the best tool for troubleshooting case 4.
Thus, depending on the type of application failure, there are two types of process memory dumps we are most interested in:
- Hang dumps / Dump of application when it is in a 'bad' state
- Crash dumps
Hang dumps are useful for troubleshooting the case of a 'hung' or 'frozen' processes. In some cases they may help to troubleshoot processes that are in an 'idle' state (process is doing nothing, although should be really busy). They also can help for investigating abnormal memory consumption or troubleshooting memory/resource leaks. Producing a hang dump is typically initiated by the user.

Crash dumps are used for identifying the reason a process crashes. Crash dumps are produced automatically by a dump creation tool once the tool is configured properly.

FAQ:
Q: How to understand the process is crashing / was crashing?
A: The easiest way is to review Windows Event logs. System or Application event log should contain related errors inside. Another way is compare process PID with initial process PID - if PIDs are different then there was a crash or the process was restarted. The last way is to review logs. Most SI components write special entries into their log file every time their process / service starts.

Q: What should I do if process is crashing?
A: Collect 'crash' dump

Q: What should I do if process is 'hung' or doing nothing at the time it should really be busy or consumes a lot of CPU when it should be idle.
A: Collect 'hang' dump

Q: What should I do in other cases?
A: Dumps are unlikely to help, but logs still can. Please collect logs.
