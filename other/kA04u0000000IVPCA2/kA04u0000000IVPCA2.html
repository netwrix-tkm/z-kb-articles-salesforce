Issue: Summary:
SI-3738 29749 - SI agent CPU usage High, followed by disk failure [ALIOR]
Behavior before Fix:
Previously it was impossible to customize EXCSIDS attribute in the FILE_MONITOR section of the SBTFileMon.ini which allows to filter out event from particular perpetrators globally on the level of driver pre-filters.
Behavior after Fix:
Now it is possible to use new configuration parameter 'fsExcludeSIDs' in the SIWindowsAgent.exe.config file. which allows you to extend list of globally excluded perpetrators by adding them to EXCSIDS attribute in the FILE_MONITOR section of the SBTFileMon.ini file. 'fsExcludeSIDs' parameter value must be added into '' part of the SIWindowsAgent.exe.config and value must be specified in the form of semicolon separated list of SIDs.
