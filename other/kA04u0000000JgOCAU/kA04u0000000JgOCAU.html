Step-by-Step Instructions:

  1. Open the detailed task view of the stuck task via the Web UI:

  2. In the Task Detail, you'll find on the top right the ID of the task. It will look something like this: "f1893e96-ee2d-42d4-ba26-05da7fe2cf47". Copy and save the Task ID. Below is a screenshot as an example:

  3. Open the currently scanned report via (if applicable, there may not be a report):
    detailed task view > Current Report (Top Left)

  4. Going into the report, you‚Äôll find the ID of the current report in the same place as the task ID in 2) Copy and save the report ID (if applicable, there may not be a report)

  5. Log in to the GSM as admin via console/SSH and enable the superuser 'root' in the menu Advanced > Support > Superuser. Set to [enabled], provide a password and Save.

  6. Open the GOS shell Menu and navigate to Advanced > Support > Shell

  7. Become the superuser by entering:
    su
    then the corresponding password

  8. Become the database user by entering:
    su ‚Äì postgres

  9. Open the database edit shell by entering:
    psql gvmd

  10. Set the Task to status done by entering
    UPDATE tasks SET run_status = 1 WHERE uuid = '';

    NOTE: replace with the ID from Step 2

  11. Set the corresponding report to "done" by
    UPDATE reports SET scan_run_status = 1 WHERE uuid = '';

    NOTE: replace with the ID from Step 4

  12. Leave the database shell with
    /q
