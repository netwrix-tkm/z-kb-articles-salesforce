Summary: How to Query File Level Details From Sensitive Content Hits

Issue: Sensitive Data Hits, File Level Details and Owner Definitions are all stored on separate tables.  This makes querying file level detail on sensitive content difficult. 

Instructions: This sql script requires you to collect File Level Details (option in FSAA scan Query).  This option is off by default.  Enabling this option will cause longer scan times, database growth, and cause your File System reports to take longer to generate.   
 
IF OBJECT_ID('SA_DLP_FileDetails', 'U') IS NOT NULL
DROP TABLE SA_DLP_FileDetails

SELECT
      m.[HostName]
      ,[CriteriaName]
      ,[FileName]
      ,[FilePath]
      ,[MatchCount]
      ,[IsExcluded]
      ,r.[DeletedUSN]
       ,[NestedLevel]
      ,[Size]/1024 AS 'Size KB'
      ,[LastModified]
      ,[LastAccessed]
       ,u.[DisplayName] as 'File Owner'
       ,[IsDeleted]
      ,[ACCOUNTDISABLE]
      ,[SID] AS 'Owner SID'
      Into SA_DLP_FileDetails
  FROM [dbo].[SA_FSDLP_MatchesView] M
  Inner Join SA_FSAA_ResourcesView R
  on m.[HostID] = r.[HostID]
  and m.[ResourceID] = r.ID
  inner join [SA_FSAA_Trustees] T
  on  r.[HostID] = t.[Host]
  and r.[OwnerID] = t.ID
  Left join [SA_ADInventory_UsersView] u
  on t.[SID] = U.[ObjectSid]
