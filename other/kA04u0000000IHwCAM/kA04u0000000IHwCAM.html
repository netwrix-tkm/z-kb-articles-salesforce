Summary:

  1. Logging service tries to send a huge message to Splunk
  2. Syslog and AMQP - New Macro for File Size (NAS)

Behavior Before Fix:

  1. Issue in syslog library that causes memory corruption and buffer file to grow large in certain cases.
  2. %FILE_SIZE% syslog macro was not working correctly for nas devices.

Behavior After Fix:

  1. Implemented fix that will deal with memory corruption in these certain cases.
  2. %FILE_SIZE% syslog macro has been fixed and is reporting file size correctly to syslog.

  1. Disable all hosts.
  2. Stop Logging Service ( sc stop SBTLoggingSvc )
  3. Check for existing syslog buffer files. If existing syslog buffer files exist, delete them. They are probably corrupt. Defualt location ( C:\Program Files\STEALTHbits\StealthAUDIT\FSAC\Data\*.syslog )

  5. Upgrade all agents.
