Support Process. For when you don't know the Answer!

There will be times during your NNT career when you are faced with either a problem you do not know the answer to or even an area of an NNT solution that you are unfamiliar with. When this happens you should consider the following:

Preparation & Approach

If you have an agreed session with a customer then do as much prep as possible, do not go into the session cold!
Install the product - Essential if you are unsure about a feature of a product. If time allows get an installation of the product in question and 'play' with the feature before any session with the customer. It also helps to have the software at hand during a session.
Review manuals - The Change Tracker manuals in dropbox contain information about most of the features within the Change Tracker product. If you are dealing with Log Tracker then the manuals can be found in the 'Home' tab of the install of within the 's-docs' folder of the installation path.
Speak to your colleagues - No one is an island, we all have strengths and weaknesses in our product knowledge which largely comes down to experience and note taking. If you are unsure and unable to figure out how something works from the manuals and testing, ask a college. Note, its best to ask for help as far in advance of working with the customer, don't leave it until you are on the session.
If you have prepared as best you can but are still unsure, or if you have been blindsided by a question on a session, try and project a confident demeanour and use terms like:
(if you have a rough idea)
My understanding of how it works is….
Perhaps we should come back to this on a later session….
Let me take this away and come back to you.
I think I'll check on that if you don't mind and come back to you with an answer.


Information Gathering

If you are going to offer to take the problem/question away then gathering as much information as possible is key. You are going to have to explain this to someone else after all.
Start with the basics, get the software version numbers and if relevant, the details of the operating system which the software is running on.
If you are dealing with a problem, i.e. the software not working as it should, gather log files. Change and Log Tracker all write to log files but also don't forget any operating system log files such as:

  • The Windows Event Viewer
  • On Linux /var/logs/messages
  • If dealing with Change Tracker on Linux then Nginx creates log files

It's also important that you gather a good description of either the problem or, if it's general feature question, what the customer would like to do.
If you are dealing with a problem then collect detailed steps of how to reproduce the problem. This is the first question you'll be asked when you try and escalate!
If you are dealing with a "can the software do this..", ensure that you have as much detail as possible from the person asking the question. Tip, it is always a good idea to echo back to the customer the question which they have asked. That way you can both be certain that you understand.
Ask the customer how he or she would like the feature to work. It's always good to collect people's point of view and they may have good ideas! Even if the software doesn't work that way, it's possible that we can adjust in the future.
