Issue: Summary: SA Crash on Scheduled Task Execution Event: When Explicit Host Lists Are Used In Sched Task, SA Writes Individual Hosts from List to Event Log
Before fix:  When assigning an alternate host list on the scheduled task and running the scheduled task, the event log will write out the contents of the host list assigned.
After fix: The event log will just write out the name of the host list assigned, not the contents.

Instructions: 1.            Unblock the hotfix zip file in the windows property dialog, if an unblock button exists there.
2.            Close all instances of StealthAUDIT (check task manager under processes for all users)
3.            Stop the "StealthAUDIT Web Server" service (if running).
4.            Run UnregisterAll.bat as an administrator in %SAInstallDir%. Ensure that SAJobEngine.dll unregistered successfully.
5.            Update StealthAUDIT.exe in the %SAInstallDir%
6.            Update all Web files in your %SAInstallDir%Web folder
7.            Update all PrivateAssemblies in your %SAInstallDir%PrivateAssemblies folder.
8.            Update all Templates in your %SAInstallDir%Templates folder.
9.            Update all SP files in your %SAInstallDir%SP folder.
10.         Run RegisterAll.bat as an administrator in %SAInstallDir%. Ensure that SAJobEngine.dll registered successfully.
11.         Restart the "StealthAUDIT Web Server" service.
