Issue:
ExchangeMailbox DC encountered an error when collecting SDD data.  Error was caused by too large of data set +1000 characters.  We now correctly truncate data that is too large for tables.  This error was only seen when "Store discovered sensitive data" check box was checked in the SDD criteria screen. 
 

Instructions:

  1. Unblock the hotfix zip file in the windows property dialog, if an unblock button exists there.
  2. Exit StealthAUDIT console
  3. Close all instances of StealthAUDIT (check task manager under processes for all users)
  4. Update all DC file in your %SAInstallDir%DC directory
