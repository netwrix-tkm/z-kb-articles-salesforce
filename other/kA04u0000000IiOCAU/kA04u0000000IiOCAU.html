Summary: The StealthAUDIT SPAA agent was not releasing memory back to the server after scans were stopped or interrupted. Also the SA console was not correctly handling communication issues causing a loop that didn't end the job. This hotfix includes changes for the agent memory consumption by resetting the SPAA SharePoint agent 5 minutes after a scan finishes. Communication issues are now properly documented in the job logs and retry logic on the SA console will attempt to reconnect to the agent at 30 second intervals for 5 minutes until ending the job.
