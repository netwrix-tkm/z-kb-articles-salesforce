Summary:

  1. I-7482 Exchange | Add support for KB5007012
  2. SI-4463 60676 - The GPO monitoring policy missing changes ( Network Security: Restrict NTLM: Incoming NTLM traffic )
  3. SI-4464 60701 - Unable to traverse OU tree to scope SI policy (Error Operation Failed: Get AD Subordinates List)

Behavior before Fix:

  1. SI Agent do not support Exchange with installed KB5007012 for 2016 CU21, CU22 and 2019 CU10, CU11
  2. Logic of translating such settings is based on analyzing mapping xml file which has a list of such settings and translation of the values to the readable format."RestrictReceivingNTLMTraffic" parameter which is responsible for 'Incoming NTLM traffic' setting was missing.
  3. AD Search is used in many different places and used, for example, for getting additional information about objects to display this in event fields or to browse (and search) AD structure when you configure policy filters in SI Admin Console. All such operations are executed on SI Agent side and by default we have 5 seconds Timeout value configured for such operations. This value is configurable inside the SI Agent config file but single value is not enough for this. Because this is important to keep relatively small value when you want to get details for single object, but can be not enough when you browse or search you AD structure and big number of objects can be returned.

Behavior after Fix:

  1. Extend Exchange signatures.
  2. "RestrictReceivingNTLMTraffic" parameter is added to the mapping files
  3. Implemented additional configuration parameter in the SI Agent config file which will be used for such browse/search requests when you configure filters in SI Admin Console. This new parameter is 'extendedDirectorySearchTimeout' with default value set to 30 seconds.
