Summary: Enhance installer options to start agent service

Issue: Summary:  Enhance installer options to start agent service
Before fix:  Agents deployed to very busy sites may cause the server to reboot during hook.
After fix:

  • New check box option in the SI console for agent deployment to allow the agent to be deployed in a disabled / stopped state.
  • New silent installer option to allow for the agent to be deployed in a disabled / stopped state
    • STARTAGENTSERVICE="TRUE" - agent service starts on installation.
    • STARTAGENTSERVICE

Instructions: SI Win Console:
1. Close the SI Win Console
2. Extract attached files
-  'Common.Tools.Deploy.dll '
-  'SIWinConsole.exe'  
 to the " C:\Program Files (x86)\STEALTHbits\StealthINTERCEPT\SIWinConsole" folder
3. Start SI Win Console
 
SI Agent:

  • Deploy Agent using SI Console
    • Extract attached 'SI Agent.exe' or 'SI Agent(signed).exe' ' to the C:\Program Files (x86)\STEALTHbits\StealthINTERCEPT\SIWinConsole\Setup' folder
      • 'SI Agent(signed).exe'should be renamed to the 'SI Agent.exe'
  • Install SI Agent manually
    • Uninstall old SI Agent (if necessary)
    • Install the attached SI Agent or SI Agent(signed)
  • Silent SI Agent installation
    • Uninstall old SI Agent (if necessary)
    • New silent installer option to allow for the agent to be deployed in a disabled / stopped state
      • STARTAGENTSERVICE="TRUE" - agent service starts on installation.
      • STARTAGENTSERVICE
