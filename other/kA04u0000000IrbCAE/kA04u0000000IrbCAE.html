Summary: StealthAUDIT Analysis Modules do not currently support Unicode characters, which prevents the utilization of SQLScripting module within the StealthAUDIT console. This prevents the ability to scope specific folder paths for the FS_ResourceBasedGroup job.

Issue:

StealthAUDIT Analysis Modules do not currently support Unicode characters, which prevents the utilization of SQLScripting module within the StealthAUDIT console. When attempting to either add Unicode characters within the interface or manually add Unicode characters to a SQL script in StealthAUDIT, the value saves but does not appropriately process them in the backend. When reopening the interface (Analysis window), the characters do not appear in originally entered form. Executing actions against paths that contain Unicode characters is supported, but filtering and targeting within the GUI is not. The GUI will show "?" in place of the Unicode characters.
 

Instructions:

StealthAUDIT 8.2 did introduce some Unicode support. For example, the TextSearch Data collector can now parse Unicode encoded csv and tsv files. With this we can configure a similar workflow that the SharePoint 5.Effective Access Audit uses for the FS_ResourceBasedGroups job.

 

Custom Resource Based Group job is also available on the FTP: http://downloads.stealthbits.com/access/files/Utilities/Jobs/RBG%20Custom.zip

See linked Word document for instructions:
https://downloads.stealthbits.com/access/files/KB_Attachments/KBArticle1.docx
