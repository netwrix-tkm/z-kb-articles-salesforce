Summary: The SQL DC licensing has changed with the 9.0 release to support the addition of new database platforms

Issue: Once the SQL Solution Set was re-released with StealthAUDIT 8.1, the SQL data collector was only laid down with a SQL specific license.
In 9.0, with the release of the Oracle Solution, the SQL DC must now be included as a Core DC due to the fact that both the Oracle and SQL License requires these dlls. This is because the same DC is used by both solutions, with additional DLLs used to distinguish what aspects of the data collector will be available based on licensing. This was done to accommodate allowing end users to scan both SQL and Oracle systems for Permissions, Activity, and/or Sensitive data with a single scan job (if they have both features properly licensed)
While the data collector component itself is laid down with core, functionality within that data collector will not work unless you have one or both of the following licenses

  • SQL
  • Oracle
The list below describes what dlls are laid down with each relevant licensable feature.
  • Core License now lays down
    • DC/DCSQL.dll
    • PrivateAssemblies/Stealthbits.StealthAUDIT.DataCollectors.SQL.dll
    • PrivateAssemblies/Stealthbits.StealthAUDIT.DataCollectors.SQL.UI.dll
    • PrivateAssemblies/Stealthbits.StealthAUDIT.DataCollectors.SQL.Common.dll
  •       SQL License now only lays down
    • PrivateAssemblies/Stealthbits.StealthAUDIT.DataCollectors.SQL.SQLServer.dll
    • PrivateAssemblies/Stealthbits.StealthAUDIT.DataCollectors.SQL.UI.SqlServer.dll
    • InstantSolutions/GROUP_SQL
  •        Oracle License (NEW) lays down
    • PrivateAssemblies/Oracle.ManagedDataAccess.dll
    • PrivateAssemblies/Stealthbits.StealthAUDIT.DataCollectors.SQL.Oracle.dll
    • PrivateAssemblies/Stealthbits.StealthAUDIT.DataCollectors.SQL.UI.Oracle.dll
    • InstantSolutions/GROUP_Oracle
