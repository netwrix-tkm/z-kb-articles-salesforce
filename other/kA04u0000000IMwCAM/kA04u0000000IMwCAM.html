Summary:  Large columns (500mb+) of certain data types were causing timeouts during SDD scans.  Also included in fix is a additional SDD setting that allows users to chose whether they scan large columns.

Before Fix:  large columns would cause the following error and interrupt the scan

"'SensitiveDataCollector' encountered an error on database 'Database'.  It will be skipped. Message: 'Exception of type 'System.OutOfMemoryException' was thrown.'"

After fix:  Some data types that are likely to cause OOM exceptions are skipped in the scan.  New table SA_SQLServer_SDD_SupportedDataTypes can be referenced to find the supported data types.  Large columns can now be scanned without OOM errors, but the user now has the ability to choose whether large columns are scanned.
