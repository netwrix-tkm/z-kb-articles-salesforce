Summary: Unix DC using hardcoded paths for scripts

Issue: Summary: Unix DC using hardcoded paths for scripts
Behavior Before Fix:  Unix DC was pre-configured to use scripts that were located under the default C: path.
Behavior After Fix:  Unix DC now allows the use of the %SAINSTALLDIR% variable so that path doesn't have to be modified if SA is installed in a non-default location.
