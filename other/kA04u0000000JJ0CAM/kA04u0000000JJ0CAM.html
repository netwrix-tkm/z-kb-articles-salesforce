Summary: SharePoint SDD Errors appearing as Criteria Names in Tier 1 DB / SPSEEK Bulk Import - Imports all match hits as dupes when a new match is found / Consecutive SharePoint SA SDD Scans skipping newly added files / SPAA Agent 8.1 high memory use

Issue: Summary:
SASP-2740 23959 - SharePoint SDD Errors appearing as Criteria Names in Tier 1 DB

  • SDD logs errors in Tier2, we used to include those errors in our bulk imports which would then get passed to the AIC and pollute the reports. We no longer import these errors but they are still written to the tier 2 sqlite files.
SASP-2891 23107 - SPSEEK Bulk Import - Imports all match hits as dupes when a new match is found
  • There was an error in the bulk import logic regarding differential scans where we would correctly add new files to tier 1 - BUT - would add ALL of the match data for previously scanned files again as new rows. Essentially ballooning the size of the table with duplicate data.
SASP-2803 23106 - Consecutive SharePoint SA SDD Scans skipping newly added files
  • We weren't correctly scanning newly added SDD files during our differential scans, now we are adding them correctly.
SASP-2785 24094 - SPAA Agent 8.1 high memory use
  • Customer was experiencing high memory use during agent scans. This was due to the way we were processing the SharePoint Hierarchy regarding list concurrency.
SASP-2594 Remove ADI Lookups from SPAA Scans
  • Removes the requirement for Port Access when running Agent based SPAA Scans.
