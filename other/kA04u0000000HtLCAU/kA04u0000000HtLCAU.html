Summary:

StealthDEFEND Not Receiving Events from SI Agents

Behavior before Fix:

Under rare conditions RMQ client could throw an exception on disposal, as the result new event sink wasn't fully created and old one wasn't fully disposed so event flow between Agent & Defend stopped till agent restart.

Behavior after Fix:

RMQ client disposal wrapped into try block to avoid the issue. Also, RMQ auto-recovery was turned off to avoid potential incompatibilities with Defend's RMQ server.
