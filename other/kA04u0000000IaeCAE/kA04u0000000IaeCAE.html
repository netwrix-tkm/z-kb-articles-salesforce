Summary: Resolves issue with SPAA Bulk Imports where Users were running into FK constraints because we were importing the RolesProxyId's for the children of a deleted Parent.

Issue: Summary:
Resolves issue with SPAA Bulk Imports where Users were running into FK constraints because we were importing the RolesProxyId's for the children of a deleted Parent.
Behavior After Fix:
SPAA Bulk imports are appropriately cleaning up RolesProxyId's for deleted items.
Instructions for applying Hotfix:
1. Unblock the hotfix zip file in the windows property dialog(Get-ChildItem -Path '' -Recurse | Unblock-File), if an unblock button exists there.
2. Close all instances of StealthAUDIT (check task manager under processes for all users)
3. Exit StealthAUDIT console
4. Copy and replace files from the DC folder in this hotfix, to the DC Folder in SA Install Dir
5. Copy and replace files from the PrivateAssemblies folder in this hotfix, to the PrivateAssemblies folder in SA Install dir.
6. (For updating Agent:) Take a backup of the files in SPAA directory on target SharePoint server
7. Uninstall existing agent
8. Install SPAA Agent 8.2.0.13
