Summary: SR Long rollback time / Rollbacks not completely restoring

Issue:

SR 1.4 HF 00138619SR-5341Long rollback time / Rollbacks not completing correctly.


When a rollback contains an object with a DistinguishedName change and other attribute changes, SR moves the object to the source OU before completing all attribute changes which causes the object to not be fully restored as it is trying to apply the attributes to the deleted items location of the object.  

This has been resolve in this hotfix, and now the attribute changes use the restored location as to be set correctly.
