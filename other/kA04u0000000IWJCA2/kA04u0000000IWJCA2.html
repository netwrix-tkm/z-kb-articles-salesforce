Summary: SI Agent may hang in 'Starting' mode for ~40 minutes after reboot

Issue: Summary:
SI Agent may hang in 'Starting' mode for ~40 minutes after reboot
Behavior before Fix:
Agent is in 'starting' state for ~40 minutes about DC reboot. Hang-up is caused by a deadlock agent is waiting for dependent services startup (sbtservice and plsahlp driver) and SCM is waiting till agent service will fully start.
Behavior after Fix:
Agent reports that it is in 'started' state to SCM before it starts dependent services
