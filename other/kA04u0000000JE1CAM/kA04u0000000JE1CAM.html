Summary: You have used the "Remove host data" category, a machine name like "!REMOVED_q3402349803480583" is displayed

Issue: You have used the "Remove host data" option in the File System Access Data Collector to try and remove host data, but the job timed out.  Now instead of the machine name you were trying to remove, you see a machine name like "!REMOVED_q3402349803480583" in the Access Information Center and other places that list hosts in the product.

Instructions:

  1. Change the job timeout to a longer value.  (Right click job, choose Properties, adjust Timeout minutes setting in General tab.)  Setting this value to 1440 minutes (24 hours) for safety should good, although the job is unlikely to actually take that long.
  2. Re-run the removal job, targeting the same hostname you previously did.  The removal job will know to associate the "!REMOVED." machine with the original hostname and both will be removed when the job completes successfully.  (This process can take a long time if there is a lot of data in the database or for the individual machine, so the recommendation in general is to change this timeout to a longer value than 20 minutes when using this utility function.)



Extra info:  How to create the "Remove host" job with the File System Access Data Collector
  1. Create a blank job
  2. Create a query that uses File System Access Data Collector
  3. Configure the query, and on the Query Selection (first page), choose "Remove host data".  [Picture below shows AUDIT 9.0 version]
