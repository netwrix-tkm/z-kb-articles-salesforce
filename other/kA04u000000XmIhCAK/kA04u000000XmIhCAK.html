The Customer Experience Program helps Netwrix improve the quality, reliability and performance of Netwrix products and services. You will be prompted to participate in this program during the Netwrix Auditor setup.
If you accept, Netwrix will collect statistical information on how the Licensee uses the product, including the metrics listed below. Data is collected in accordance with the applicable law.
 

Collected metrics

Usage statistics, including the following numbers:
  • Alerts (configured and triggered)
  • Subscriptions to the reports (including Enterprise Overview dashboard) and saved search queries
  • Subscriptions to the risk assessment results
  • Monitoring plans
  • Saved search queries
  • Exports performed
  • Role assignments and delegations
  • Errors and warnings in the Health log
  • Collected state-in-time snapshots
  • Collected Activity Records by data source
  • Monitored items by data source
  • Effective omit settings by data source
  • For Exchange Online and Azure AD –  number and types of Mailboxes and Users

Operations data, including:
  • Data collectors’ activity periods by data source
  • Netwrix Auditor Client sessions:
    • Type - remote or local
    • Screen resolution
  • Netwrix Auditor Client host:
    • OS type and version
    • CPU architecture
  • License counters
  • Risk Assessment configuration settings

User experience, including the following numbers:
  • Transitions between UI windows
  • Health Status views
  • Executed search queries
  • Viewed reports
  • Viewed reports on Risk Assessment by group

Capacity, including:
  • Netwrix Auditor Long-Term Archive size and free space on the drive where it is stored
  • Number of files and folders processed by the File Server collector (FSA) and their total size

How it works


A UsageStatistics record is created by Netwrix Auditor on the corresponding occurrence - for example, when a number of Activity Records is produced by data collection session for a data source.

This record is sent to the API endpoint in JSON format and stored in the stats database for analysis.

UsageStatistics record example:

"DataSource":"15ca5191-0edf-4f4e-8261-0b539fee12d3",
