Summary: Shows how to set the working directory to a folder on another volume by specifying additional parameters

Issue: When changing directory from one to drive to another in command prompt windows does not change the working directory.  Instead, you need to first change the drive the directory is on first, which can be annoying (or unknown).  For example open command prompt it will take you into the users working directory typically C:\Users\username - now type cd E:\somefolder on E: drive it will show C:\Users\username until you type the drive letter - in this example, E:
 

Instructions: open command prompt type cd /d [path you want to enter].  Note you can use windows environmental variables here also. The /d switch tells CMD that you're specifying a different drive than the current one.
 
Example:

  • Not working: Open command prompt on a server that has StealthAUDIT installed in a different drive to C:\, type in cd %sainstalldir% and press enter. You will not that it reverts to working directory of the users profile.  Close command prompt.
  • Working: Open command prompt on a server that has StealthAUDIT type in cd /d %sainstalldir% and press enter. You will now see it change to the correct directory
 
Notes:
 
  • You can also simply type the drive letter of the destination directory, which will set the working directory to the root of the volume. From there, the cd command no longer requires the /d switch.
  • Windows PowerShell handles this without the need to leverage the /d parameter, since cd in PowerShell is actually an alias for the Set-Location cmdlet, which is completely different (though similar) to the cd command in the standard Windows Command Prompt.

Versions: Windows 1.0 and later
