Summary: Consecutive SharePoint SA SDD Scans skipping newly added files

Issue: Summary:
SASP-2803 23106 - Consecutive SharePoint SA SDD Scans skipping newly added files
Behavior Before Fix:
The customer was receiving an error: Agent Polling Fault: System.ServiceModel.Dispatcher.NetDispatcherFaultException

WPF is the remoting technology SPAA uses to talk between agent and SAconsole. The MaxStringContentLength setting configures one of it's internal components which is used for reading serialized objects which are passed back and forth between agent and console. It has a default setting of 8k (which we don't alter), which limits the maximum size of an object passed back and forth. It looks like the customer has an object which is larger than that. To fix this we should increase the setting from it's default value. This can be done via the config file(s).
Behavior After Fix:
The default value was extended.

Instructions: 1. Unblock the hotfix zip file in the windows property dialog, if an unblock button exists there.
2. Close all instances of StealthAUDIT (check task manager under processes for all users)
3. Exit StealthAUDIT console
4. If the installed version of SDD is lower than SDD 8.1.0.34 - Uninstall SDD add on from SA Console
5. Navigate to Sensitive Data Folder in SA Install Dir > there may be leftover files that need to be cleaned up prior to installation of new sdd package.
6. Copy and replace the StealthAUDIT.exe.config file in the root folder of the SA Install Dir
7. Copy and replace files from the DC folder in this hotfix, to the DC Folder in SA Install Dir
8. Copy and replace files from the PrivateAssemblies folder in this hotfix, to the PrivateAssemblies folder in SA Install dir.
9. Install included SDD package on the SA Console(x86)
10. Take a backup of the files in SPAA directory on Target SharePoint server (in case the customer has config files that have been customized)
11. Uninstall SDD Add- on from target SharePoint Server(s), delete the contents of the SensitiveData folder
12. Stop the Agent Service
13. Copy and replace the contents of the included Agent Folder into the SPAA directory
14. Restart the service
