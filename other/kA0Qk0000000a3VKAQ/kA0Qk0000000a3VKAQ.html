Symptom

The policy requirements are not visible in Netwrix Password Policy Enforcer/Web (PPE/Web) clients.

Causes

  1. The Only accept encrypted client requests option is enabled.

  2. The [LIVE_POLICY] declaration is specified instead of [POLICY] in the Policy Properties Messsages tab.

Resolution

Refer to the following steps:

  1. Disable the Only accept encrypted client requests option in PPE.

    1. In the left pane of the main PPE window, select Password Policy Server and click PPS Properties.

    2. In the General tab, uncheck the Only accept encrypted client requests checkbox and click OK to save your changes.

  2. Specify the [POLICY] declaration.

    1. In the left pane of the main PPE window, extend the Policies node, right-click the target policy, and select Properties.

    2. In the Messages tab, replace the [LIVE_POLICY] declaration with [POLICY].

    3. Save the changes by clicking OK.
