Summary: Summary: Job Properties: Fix Auto Retry Functionality Issue/Request: Auto-Retry feature does not create a scheduled task to retry the task when the host status returned is "Error" as opposed to "Errors", same case for "Warning"/"Warnings". The status changes are due to an updated SAFixHostStatus store procedure. Fix/Enhancement: Now Auto Retry checks for both occurrences of each of those statuses in order to create the scheduled task.

Issue: Summary:
Summary: Job Properties: Fix Auto Retry Functionality
Issue/Request: Auto-Retry feature does not create a scheduled task to retry the task when the host status returned is "Error" as opposed to "Errors", same case for "Warning"/"Warnings". The status changes are due to an updated SAFixHostStatus store procedure.
Fix/Enhancement: Now Auto Retry checks for both occurrences of each of those statuses in order to create the scheduled task.
