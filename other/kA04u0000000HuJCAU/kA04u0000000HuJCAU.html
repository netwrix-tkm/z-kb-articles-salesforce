Summary:

  1. SQL - Certain events not being reported
  2. SAM not collecting events from DC with updated SI agent
  3. ini is available to unprivileged users
  4. SAM is not receiving activity from SP online - Failed to authenticate Tennant
  5. 2019 Server | Kerberos events may be lost with Out-of-band KB5008602, published 11/14/2021
  6. SUVP 21-11 | Kerberos, DCsync or LDAP events may be lost or incomplete after applying MSFT updates, release: 11/09/2021

Behavior Before Fix:

  1. For SQL, SAM did not report grant / revoke permission events.
  2. Upgrade from 4.1 to 6.0 hung; changing agent's safe mode settings did not work in console, multi-select agent uninstall resulted in console crash.
  3. ini inherited permissions from the parent folder. As a result, any user could read it.
  4. SAM Sharepoint Online monitoring did not support HTTP Proxy.
  5. Kerberos events may be lost or incomplete from 2019 Server with installed KB5008602.
  6. Kerberos events may be lost or incomplete with installed KB5007260, KB5007247, KB5007192, KB5007206.

Behavior After Fix:

  1. SAM now reports grant / revoke permission events for SQL.
  2. SAM is now be able to change agent's safe mode settings correctly; upgrade from 4.1 shouldn't cause a hang anymore; and multi-select uninstalls shouldn't crash console anymore.
  3. The console replaces agents.ini's security descriptor, disables inheritance, Full control to SYSTEM, Full control to BUILTIN\Administrators, Raises the integrity level to High. As a result non-admins or Medium and Low integrity processes cannot read/write the file.
  4. SAM added agent settings for HTTP Proxy support. The proxy will be used by Azure AD and SPO collections.
  5. Signatures updated; SI Agents must be fully functional.
  6. Signatures updated; SI Agents must be fully functional.
