Summary: SI Agent stopped working due to LDAP query attempts against 127.0.0.1

Issue: Summary:  SI-3290 16988 - SI Agent stopped working due to LDAP query attempts against 127.0.0.1
Before fix: agent could incorrectly determine that machine is NOT DC while it was DC or vice versa. This could lead to incorrect usage of 127.0.0.1 inside ldap queries leading to failures.
After fix: Mechanism of detection DC/non-DC has been reworked in several areas:
  -  Now all the code uses the same approach for detection
  - Detection function consists of two approaches, if first one fails we give another try using WMI-based approach
  - Now we properly handle case when detection DC/non-DC failed so we don't know what this host is. In such a case 127.0.0.1 is NOT injected into LDAP prefix.
