Summary:

Issue: The most common issue in Survey implementation is configuration for the Web interface to locate the SA Database.

If the database is on a different machine than the Tomcat Server (SA Console) the Apache Service must have sufficient credentials to access the SQL Server.

The Service will use the default Storage Profile to obtain credentials. It locates the SPProfiles by looking up the SA InstallDir in the registry. If Windows credentials are specified in the profile and Apache is running under a local system account it will fail to connect. Either change the Service to run under Domain credentials or use SQL credentials in the default Storage Profile.

Instructions:

These are the Survey test steps performed during the action configuration and what to look for when they fail:

Create DB Tables 

This failure is most often due to the SA Install path registry entry being invalid.  

Deploy to Web Server

The path to the SA Reporting Service is incorrect or files have been marked as "read only" in the SA Dashboard folder structure. When a survey is created all of the Web parts are moved to this location by the Wizard. This error usually requires a stop of Apache to fix and then restart.
 
SMTP connection
The SA Notification settings are in error. The "send from" address must be in the local domain or the test will fail (although Execute will actually run). Another possibility is that the email targeting property selected in the Wizard is invalid.
 
DB Connectivity on Web
This occurs when the DB is on a separate machine and SA is configured to use Windows authentication. To fix either change to SQL authentication or change the Apache service to run under a Domain Admin account. Then delete the survey in error and re-create it. This may also be caused by Tomcat Service being down.
If you are running VMWare Server in your console, you need to stop the 'VMware Server Web Access' service to be able to start Apache TomCat.
 

Other:

The java path has to be added to the 'Path' Environment Variable. Example: in a 64 bits machine the path is usually:

C:\Program Files (x86)\Java\jre6\bin

Add the path at the beginning of the string and separate from the rest with a ;.
