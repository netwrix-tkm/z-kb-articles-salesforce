Summary: SI Web Reports allows you to Delete a Public Report; SI Web Report Public Parameters are Global, not by user or session; Favorite report disappears after pressing "refresh" button

Issue: Summary:  

  • SI-3220 13196 - SI Web Reports allows you to Delete a Public Report
  • SI-3221 13197 - SI Web Report Public Parameters are Global, not by user or session
  • I-1625 Favorite report disappears after pressing "refresh" button
Before fix:
  • User is able to delete the Public report as a non-owner.
  • When Parameters are changed on a public report, they are set Globally. This causes others who are viewing the report to get unexpected results if they Refresh their view of the report.
  • Favorite report disappears after pressing "refresh" button
After fix:
  • Deletion of Public report is available only for report owner and for Administrator (Administrator role in Win Console).
  • Parameters configurationfor public report can be changed globally only by:
- Report owner (report user role in Win Console)
- Administrator (administrator role in Win Console).
For these users changes in report configuration are saved in DB. One instance of parameters for one report.
All other users have local configuration of parameters inside Web App session. These parameters are not saved in DB and expire after WR user session is closed (or by 3 day timer). One instance of parameters per report per user.
  • Favorite reports are stored after Refreshing

Instructions: 1. Stop services:
                - SIReportingService
                - SISchedulerService
2. Stop IIS.
3. Replace by attached files:
- SIReports\ReportingConsole\ClientBin\SLReportingPages.xap
 - SIReports\ReportingServices\SI.Services.Reporting.Implementation.dll
4. Start IIS.
5. Start services:
                - SIReportingService
                - SISchedulerService
6. Clean all browser history  (IE -> Internet options - > Browsing history)
