Issue: Summary:
Attempts to start a SQL Sensitive Data scan on a SA console with restrictive UAC policies was resulting in the scans not starting because the process was unable to create an SDD folder under the StealthAUDIT\Jobs\SA_CommonData\SQLDC directory.

Related Errors:
Failed to initialize new scan session: Unknown error
Access Denied for folder SQLDC
With this fix the StealthAUDIT should be able to create the SDD folder and the scan runs successfully.
