Issue: Summary:
This hotfix adds retry logic to the SQL SDD scan to alleviate timeout and disconnect issues.
Behavior Before Fix:
SQL SDD scans did not handle temporary disconnects or server side timeouts and scans could be interrupted.
Behavior After Fix:
Retry logic will now fire when StealthAUDIT sees a server side disconnection.  The debug logs will now indicate that connection attempts are being retried, for example "Retrying connection attempt '1' of '3", and the SQL SDD scans should now be more resilient to any possible environmental issues that would interrupt it before the fix.
