Issue: Disable logging for PS DC

More explanation of the fix:

The setting added for this is called UniqueName. If you save a config you will see it in the XML.

It is difficult though since the PowerShell DC saves its config values as an XML block inside a XML block. So the

UniqueName should be the name of the folder (not the path) you want the log files to be saved under. If your UniqueName is TestUnique, the log files will be saved under Jobs\SA_CommonData\PowerShell\TestUnique. There is no easy way to completely disable these files. Only up to 3 files will ever be created in the UniqueName folder, the emergency-log.txt, startup-log.txt and bootstrap-log.txt in certain conditions. Any log file collisions between processes will be safely ignored.

(The instructions below should help with what needs to be updated in the xml)

Instructions:

  1. Unblock the hotfix zip file in the windows property dialog if an unblock button exists there.
  2. Close all instances of StealthAUDIT (check task manager under processes for all users)
  3. Exit StealthAUDIT console
  4. Copy and replace files from the folders in this hotfix to the corresponding folders in SA Install dir.
  5. Open StealthAUDIT console and browse to the PowerShell job you wish to reduce logging on
  6. Open the job XML and looks for:

              

(If this does not appear in the xml try saving and reopening it)
