FIRST STEP CHANGES (DEMO SERVERS)

On all 3 servers, do the following one by one:

  1. Open Security Policy > Account Policies > Password Policy > Double Click "Enforce Password History" and set the setting from '10' to '24'
  2. Open up the browser and go to the CT Console, and go to the devices page, and start the 'Security Policy' Tracker.
  3. Wait for the event to come through, if any other events come through, manually acknowledge them. For the password history event, manually acknowledge it to the "Planned Security Policy Changes" planned change.
  4. Go to Settings > Groups > Windows > Server 2016 > Scheduled Reports > Edit the "Demo_NNT_CIS_Microsoft_Windows_Server_2016_Benchmark_v1.0.0 - Level 1 Member Server" and setSchedule (minutes) to '1'
  5. While the reports run, you can continue to make the other changes. We are doing this so that we erase all other reports in the past so that we can get 100% accross the board
  6. Repeat these for the other 2 Demo systems. Once these reports are set to run, proceed to making other changes as they run in the background. Start with Linux Server Changes.

THIRD STEP CHANGES (ROUTER CHANGES)

To replicate changes for the NNT Router, do the following one by one:

  1. Open mRemoteNG from Demo1 and double click the NaplesR1 Router from the left side. You only need to connect to this once, unlike the linux servers, this device is agentless and all 3 servers can use the same one device.
  2. You will be prompted for a username and password.
    1. U/N: root
    2. P/W: passWord121
  3. Type "enable" and hit enter. You will be prompted for the enable password
    1. Enable Password - passWord121

  4. Copy and Paste the following Configurations. You can copy and paste the entire block, you do not have to do it one line at a time. afterwards.

    conf t
    banner motd " Welcome to the NNT Router "
    username hacker password password
    int loopback 1
    ip address 192.168.15.1 255.255.255.0
    exit
    ip route 192.168.15.0 255.255.255.0 loopback 1
    exit

  5. Go to Demo 1, start the Process Output tracker for the router, repeate for demo 2 and 3.
  6. Wait for the events to come through, you may have to refresh the events page. It may take a minute or two for the event to come through on all 3 servers.

  7. Once it has come through on all 3 servers, go back to the router and copy and paste the following configuration:

  10. Manually Acknowledge the two events on each server into the "Planned Router Changes" Planned Change.


FINAL STEP CHANGES (DEMO SERVERS)

On all 3 servers, do the following one server at a time:

  1. Go to the Devices Page, and check that at least 7 reports have run with a score of 100% for the Windows Server. Once confirmed, continue below:
  2. Open Security Policy > Account Policies > Password Policy > Double Click "Enforce Password History" and set the setting from '24' to '10'
  3. Go to the CT Console, and go to the devices page, and start the 'Security Policy' Tracker.
  4. Keep an eye on the bottom of the page under device complaince and refresh it several times until you see a report that comes in with a score less than 100%.
  5. The second you see a the report show below 100%, Go to Settings > Groups > Windows > Server 2016 > Scheduled Reports > Edit the "Demo_NNT_CIS_Microsoft_Windows_Server_2016_Benchmark_v1.0.0 - Level 1 Member Server" and setSchedule (minutes) to 'Weekly'
  6. Wait for the Security policy event to come through, if any other events come through, manually acknowledge them into the manually acknowledged planned change rule. For the password history event, manually acknowledge it to the "Planned Security Policy Changes" planned change.
  7. Repeat these for the other 2 Demo systems.
