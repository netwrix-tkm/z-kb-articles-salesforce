Issue: Summary: One event sends multiple alerts to Splunk SIEM
Behavior before Fix:
When event is sent to SIEM it will be sent with several SIEM messages depending from the number of attributes in this event. It sends one message per attribute.
Behavior after Fix:
To simplify grouping of messages per event there is new replacement token implemented in SIEM message template: %EVENT_GUID%
When generating SIEM message it will replace this token with GUID (Global Unique Identifier) generated per event.
Example of usage:
1 %SYSLOG_DATE% %HOST% %PRODUCT% - %EVENT_NAME% - GUID="%EVENT_GUID%" PolicyName="%SETTING_NAME%" Domain="%EVENT_SOURCE_NAME%" ...
