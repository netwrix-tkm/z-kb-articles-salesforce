Summary: AIC inactivity timeout loading scan times during review creation

Issue:

  • Issue: After selecting 500 resources, the AIC was timing out from inactivity after 15 minutes while waiting for the bottom of the page to load the scan times for each resource.
  • Fix: Performance for data loading and review creation was improved, and also session timeout won't happen while actively loading data. Also a script to add an index in the resources table needs to be run in the database, see instructions below.

Instructions:

  1. Uninstall current version of AIC if it is already installed
  2. Install version of AIC included with this hotfix
  3. Run the following SQL script in the database: (already implemented in 8.0+)
        CREATE NONCLUSTERED INDEX [SA_FSAA_Resources_ParentResourceID_Name_IDX] ON [dbo].[SA_FSAA_Resources]
        (
                        [HOST] ASC,
                        [ParentResourceID] ASC,
                        [Name] ASC
        )
