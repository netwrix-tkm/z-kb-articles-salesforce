Summary:  FS SDD Scan - Temp Table problem

Before Fix: When running SDD scans against multiple hosts simultaneously, it is possible that some hosts will error out with the following error: There is already an object named '##SubjectProfileTempSourceIDs_8D8CDB44A122B35' in the database. This was due to the fact that SA was using a timestamp to generate the temp table name which made it possible for multiple hosts to attempt to create the same temp table.

After Fix: SA will now use a unique GUID value to use in the temp table name to avoid any potential clashes with other hosts attempting to create the temp table at the same time.
