Summary: Engineers should follow specific steps to ensure all possible troubleshooting has been performed prior to involving development in a customer issue If after the necessary steps have been taken, there is still no resolution, engineers must at least ensure to collect the necessary log files/scan artifact. This article describes the various artifacts and their locations

Issue: Often times, escalations tickets are entered without the proper artifacts necessary for development investigation. If error occurs, follow the guidelines below to not only collect the necessary artifacts, but review them to avoid unnecessary back and forth between engineering and development.
 

Instructions: Data collector issues: The following list includes all FSAA scan types, and the necessary artifacts to collect/review when issues are encountered:

Pre 9.0

  • FSAA
    • Scan Job Log: %SAINSTALLDIR%\Jobs\[Path to Job]\Output\[Job Name]_log.tsv
    • Applet Log: %SAINSTALLDIR%\FSAA\[ScanType]_Date_*.log
    • Trace Log: %SAINSTALLDIR%\FSAA\StealthAUDITRPC_%_[SCANTYPE]_*
  • FSAC
    • Scan Job Log: %SAINSTALLDIR%\Jobs\[Path to Job]\Output\[Job Name]_log.tsv
    • Applet Log: %SAINSTALLDIR%\FSAA\[ScanType]_Date_*.log
    • Trace Log: %SAINSTALLDIR%\FSAA\StealthAUDITRPC_%_[SCANTYPE]_*
    • Suspend.xml: %SAINSTALLDIR%\FSAA\[HOST]_FSAC_Suspend.xml
    • SBTFileMon.INI file: C:\Program Files\STEALTHbits\StealthAUDIT\FSAC\SBTFileMon.ini
    • Activity Log File**
  • SEEK
    • Scan Job Log: %SAINSTALLDIR%\Jobs\[Path to Job]\Output\[Job Name]_log.tsv
    • Applet Log: %SAINSTALLDIR%\FSAA\[ScanType]_Date_*.log
    • Trace Log: %SAINSTALLDIR%\FSAA\StealthAUDITRPC_%_[SCANTYPE]_*
    • SDD Logs: %SAINSTALLDIR%\FSAA\logs
  • Bulk Imports
    • Bulk Import Job Log: %SAINSTALLDIR%\Jobs\[Path to Job]\Output\[Job Name]_log.tsv
    • Tier 2 database files: %SAINSTALLDIR%\FSAA\[HOST]_*.fsaadb
    • Export of T1 temp tables

Solution Set Issues: Troubleshooting Solution Set jobs is a bit more difficult, because the job log does not contain the degree of information it does for data collection. It is important to clearly identify the issue that's being experienced, as well as to provide as much information as possible. The following list describes the information that should be included to properly qualify a file system solution set issue:

  • Scan types being executed
  • Scope of scans (e.g. Permissions scan collecting file level details, number of hosts, etc)
  • Relevant Row Counts (Resources, ActivityEvents)
  • Relevant data points (screenshot or export if possible)
  • Specific failed analysis task(not just the job)
