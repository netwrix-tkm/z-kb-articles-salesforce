Issue: Summary:  Enterprise Manager consuming almost the entire CPU resources
Before fix: Big amount of concurrent registering agents but with fast db can lead to consuming a lot of CPU.
After fix:
Refactor ServerCollection class to use on thread for all db operations related to updating changes in ServerConfigs, thus cpu is not spent on useless db updates.
 Avoid known dc updates if they are already present in the list
Fix agent state update
 As EM is the only writer to the Server table, remove update logic, thus simplifying db synchronization with table refresh with default period of 5 min. Can be tuned using serverCollectionDbSyncPeriod option in the format 00:05:00.
