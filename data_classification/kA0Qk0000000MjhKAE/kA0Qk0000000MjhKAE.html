Symptoms

A large directory was removed from the Sources and its path was excluded from future scans. However, clicking the button to expunge doesn't do anything. All files are stored in the Deleted Automatically section.

Cause

A bug that affects Data Classification v.5.6-5.7.

IMPORTANT: The identified issue is set to be resolved in version 5.8. Once this version is released, the build number will be promptly updated on the My Products page: https://www.netwrix.com/my_products.html.

Workaround solution

On a Data Classification SQL server, run the following query:

USE 'database name' 
WHILE 1=1 
BEGIN 
DELETE top (5000) FROM Pages 
where Status = 9;
                
if @@ROWCOUNT<1 break; 
END

It will remove all automatically deleted entries in the DB.
