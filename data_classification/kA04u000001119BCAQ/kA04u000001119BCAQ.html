Symptom

Data Classification cannot send requests using non-standard port (default is 3306) for MySQL source.

Cause

It is a known issue for Data Classification v5.7. It will be fixed in the next release.

Resolution

Create a port forwarding rule in one of the following ways:

  • Configure a rule on an NDC server.

    Use NDC server as target. Apply the following OS Windows CMD command line:

    netsh interface portproxy add v4tov4 listenport=3306 listenaddress=NDCSERVERIP connectport=MYSQLPORT connectaddress=MYSQLIP

    TIP: Use NDCSERVERIP as a source

    IMPORTANT: Only one listener can be added for one port in windows, so you will not be able to add more than one MySQL source

  • Configure a rule on the target MySQL server.

    For Windows based MySQL you can use netsh.

  • Configure a rule on a network device.
