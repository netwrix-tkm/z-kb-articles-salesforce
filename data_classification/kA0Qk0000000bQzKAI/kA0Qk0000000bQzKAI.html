Netwrix Data Classification Glossary

To understand folder / file structure of Netwrix Data Classification or to better communicate with Technical Support, it's important to know Netwrix Data Classification acronyms and initialisms.

General Terms

Crawling: the process of systematically scanning through an organization's data repositories, such as file servers, databases, and cloud storage, to identify and catalog the types of data present. This process involves analyzing the content of files and documents to determine their sensitivity, importance, and relevance to the organization's policies and regulatory requirements.

Content tagging: the process of attaching metadata or labels to digital content to describe its characteristics, attributes, or context. These tags provide additional information about the content, making it more searchable, discoverable, and manageable.

Netwrix Data Classification Sources

  • Box: a source of data provided by or stored within Box, which is a cloud-based content management and collaboration platform.

  • Dropbox: a source of data provided by or stored within Dropbox, which is a cloud-based file storage and collaboration platform.

  • Exchange Mailbox: a user's email account hosted on Microsoft Exchange Server or Microsoft 365 (formerly Office 365).

  • Exchange Server: the on-premises or cloud-based email server solution provided by Microsoft.

  • Content Server: a centralized repository or platform used to store, manage, and share digital content within an organization.

  • Google Drive: a cloud-based file storage and synchronization service provided by Google.

  • File System: the hierarchical structure used to organize and store files on a computer's storage devices.

  • Outlook Mail Archive: the storage location where emails are archived within Microsoft Outlook.

  • SharePoint: a deployment model where the SharePoint platform is installed and managed within an organization's own data center or private infrastructure.

  • SharePoint Online: a web-based collaboration platform developed by Microsoft.

  • Single Database: the Database source configuration screen allows you to enable the crawling and classification of content stored in your Microsoft SQL Server, MySQL, and Oracle databases.

  • MS SQL Server: a relational database management system developed by Microsoft.

Netwrix Data Classification Components

  • Source: the location or repository from which data is collected and analyzed for classification purposes.

  • Taxonomy: the hierarchical classification framework used to organize and categorize data based on predefined criteria.

  • Workflow: a predefined sequence of steps or actions that govern the process of classifying and managing data.

Auxiliary Components

  • SQL Server - Hosts databases used to stored and query audit data.

  • OCR: Optical Character Recognition. A technology that converts different types of documents into editable and searchable data.

  • DQS: Distributed Query Server. A component of a distributed database system that manages queries across multiple nodes or servers.

  • DSAR: Data Subject Access Reviews. A legal right granted to individuals under data protection regulations.

  • MIP labels: Microsoft Information Protection labels.

  • NLP Entities: specific pieces of information or objects that are extracted from text using Natural Language Processing (NLP) techniques.
