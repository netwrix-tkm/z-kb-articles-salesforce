Collector Threads slider

The Collector Threads slider allows administrators to change the number of background processes, or threads, occurring on the Netwrix Data Classification server. This slider should be left at the default of 24.
Values for threads can be set via NDC Management Web Console: http://localhost/conceptQS/Configuration.

NOTE: If you've noticed a decrease in processing capacity, it is recommended to set the Collector Thread slider to 0.

Automatic Threading

Setting the slider to 0 will switch an auto-detect mode. It will allow to detect allocated cores and set the slider to match available resources. Refer to the following article for additional information on hardware requirements: Netwrix Data Classification — Hardware Requirements.

NOTE: Once the thread slider is set to a different value, it is recommended to restart the corresponding service via Windows Services Manager (e.g., NDC Indexer service to be restarted in case the number of Indexer threads were changed, etc.).
