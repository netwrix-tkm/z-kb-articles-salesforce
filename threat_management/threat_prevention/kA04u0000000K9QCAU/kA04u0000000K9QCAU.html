Summary:
100407: SI - Powershell Connect-SIEnterprisemanger fails
102818: "Supplied certificate does not match expected certificate" errors in the Agent log after re-deploying it to the host
103049: Can't export policies in WinConsole when there are some corrupted encrypted values for Email credentials setting
Behavior before Fix:
100407: This fix is unrelated to the original issues. Original issue was caused by wrong set of certificate files in the CertsInfo folder. But after we found a workaround for this certificates related issue we observed that second call to Get-SIAgentStatus PowerShell function results in error.
102818: Previously if some SI Agent already has pinned certificate thumbprint in the database and for some reason this Agent must be re-enrolled and re-pinned - then value from database will override new value and Agent will lose possibility to connect to the EM.
103049: Previously if values for SMTP credentials setting can't be decrypted it will generate an exception which will interrupt policy export process.
Behavior after Fix:
100407: After some investigations we found that this happens when there are some entries in the Server table which point to the server for which we can't get IP Address information. Now code is modified in the way to respect empty value for such servers in both cases: when we do DNS lookup and when we re-use previous DNS lookup results from our internal DNS cache.
102818: Now new thumbprint value has higher priority than value in the database.
103049: Now the code is modified to return original encrypted value and don't interrupt policy export process
