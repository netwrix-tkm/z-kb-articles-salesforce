Because of its purpose and capabilities, StealthINTERCEPT should be operated at the same security level as the domain controllers it protects. This collection of best practices is intended to supplement customers' own security judgement and highlight risks and recommendations.

Best PracticePriorityDescription
Restrict access to the StealthINTERCEPT application serverHighIndividuals with administrative access to the StealthINTERCEPT server are implicitly administrators of StealthINTERCEPT and through its capabilities may be able to become administrators on domain controllers. Limit access to the least possible.
Operate StealthINTERCEPT at the same security level as you operate domain controllersHighThis one principle should guide decisions made about the management and operation of StealthINTERCEPT. Because StealthINTERCEPT protects domain controllers, its implementation should be as protected as they are. Consider deploying StealthINTERCEPT within the same credential boundary as domain controllers to limit the potential for privilege escalation.
Protect StealthINTERCEPT's SQL Server operating system and databasesHighStealthNTERCEPT's SQL Server database contains sensitive configuration data. The same controls applied to the StealthINTERCEPT server should be applied to the SQL server as well.
Use the Protected Users group to secure accounts that login to the StealthINTERCEPT serverMediumAdding users to the Protected Users group in active directory automatically protects the account by forcing it to only use secure authentication techniques
Periodically review StealthINTERCEPT console permissionsMediumPeriodically review holders of the StealthINTERCEPT Administrators and Console Operators roles to ensure only authorized individuals hold those roles.
Protect, randomize, and periodically rotate Windows local administrator account passwordsMediumLocal administrator credentials are frequently targeted by attackers, whether through password guessing or pass-the-hash attacks. Ensure these credentials are randomized, stored securely, and periodically changed using a solution like Microsoft LAPS.
Ensure Microsoft security updates are applied to the StealthINTERCEPT serverMediumRapidly applying Windows security updates is essential to protecting the StealthINTERCEPT server from the exploitation of known vulnerabilities.
Install and regularly update antivirus and endpoint protection productsMediumAntivirus and endpoint detection and response solutions help protect servers and applications from malware and adversarial techniques. Install your standard solutions and ensure they are regularly updated.
Install only required applications on the StealthINTERCEPT serverMediumInstalling unnecessary software on the StealthINTERCEPT server increases its attack surface and the amount of threats that must be defended against. Only install absolutely necessary software.
Ensure only server administrators can access the StealthINTERCEPT installation directoryMediumBy default, StealthINTERCEPT uses the permissions granted to program files which only allows SYSTEM, Administrators and, when windows authentication is used, the service account will be applied directly with full control and other accounts. Granting additional privileges creates additional risks to StealthINTERCEPT and should be avoided.
Use a Privileged Access Management (PAM) solution to protect administrative access to StealthINTERCEPTMediumA PAM solution allows administrators to securely connect to StealthINTERCEPT and can provide an audit trail for activities taken on the server and within the product.
Ensure Network Level Authentication (NLA) is enabled on the StealthINTERCEPT serverLowNetwork Level Authentication improves the security of the Remote Desktop Protocol connection used to administer the StealthINTERCEPT server.
