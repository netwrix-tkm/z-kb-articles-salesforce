Overview

This article covers the ManagedObjects node legend in the 1Secure Configuration.xml file. This article also includes an example of the ManagedObjects node. Refer to the following Configuration.xml location: 

    C:\ProgramData\Netwrix Cloud Agent\AgentCore\ConfigServer

1. ManagedObjects legend

The ManagedObjects node allows you to determine the source configuration applied. Refer to the following subnodes:

  • ManagedObject. A connector configuration.

  • AuditedSystems. A connector-specific value, refer to the possible GUIDs:

    • da7c0b3d-63ad-ff26-fe75-ac0712a1835d. Active Directory connector.

    • 7e498d0b-3592-4d0e-9629-c16be845f4e0. Logon Activity connector.

    • 850fe1da-0ea6-c1a8-9810-0c1cece30698. File Server connector.

  • AutoAudit. The Adjust audit settings automaticaly option. Available in the 1Secure UI.

  • DataCollectionPeriodInSeconds. The amount of time between data collections. It is set to 10 minutes by default. Unavailable in the 1Secure UI.

  • Agents. The Network compression option. Available in the 1Secure UI.

  • ScopeItems. The source configuration.

  • Reference. An internal unique identifier for the source configuration—used to identify the source in 1Secure.

  • MonitorHiddenShare. Option to audit hidden shares.

  • audit_item_value. The monitored source.

  • AccountInfo. Credentials for the crawling account.

  • CollectedActions. Actions collected in the file server.

  • OmitObject. Options to filter activities. These depend on the 1Secure configuration.

NOTE: Some subnodes are missing from the legend—they do not play a major part in the troubleshooting process. They are inherited from Netwrix Auditor.

1.1 Outdated attributes

In case ManagedObjects attributes are outdated or inconsistent, refer to the following steps:

  1. Stop the main agent service—CloudAgentCoreSvc.

  2. Delete the Status.xml file. It is located in C:\ProgramData\Netwrix Cloud Agent\AgentCore\ConfigServer.

  3. Start the main agent service—CloudAgentCoreSvc.
