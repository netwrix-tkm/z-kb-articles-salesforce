Summary

If an HTTP Error 500.19 is displayed when navigating to SbPAM in a web browser after an install or upgrade, take the following steps to resolve the error.

Instructions

  1. In Windows on the SbPAM server, open the Start Menu and search for Apps & Features.
  2. Navigate or search the list of applications until Microsoft .Net Core - Windows Server Hosting is located.
  3. Click on this application, and click Modify.
  4. In the dialog that pops-up, click Repair.
  5. During the repair, there may be a prompt to locate WindowsServerHostingBundle.exe. The folder Windows opens for this prompt should include a dotnet-hosting-<version>-win.exe file, which can be double-clicked to continue the repair.
  6. After the repair completes, reboot the server as prompted.
  7. After the reboot, SbPAM should be available via a web browser without experiencing an HTTP Error 500.19.
