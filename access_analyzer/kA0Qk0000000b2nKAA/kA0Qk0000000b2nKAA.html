Symptom

The Extractor logs significally grows each time SDD runs for a specific host. This is causing disk space to fill up and attempts to reduce logging levels for these files have been unsuccessful. The logs are primarily comprised of INFO level entries.

Cause

The root cause of this issue is the default logging level configuration in Netwrix Enterprise Auditor, which generates large Extractor logs containing a significant number of INFO level entries during SDD operations.

Resolution

To resolve the issue, follow these steps:

  1. Access the Netwrix Enterprise Auditor server.

  2. Open the 1-SEEK System Scans query properties.

  3. Hit View XML to view the XML configuration of the query.

  4. Locate the section around line 305 of the XML configuration.

  5. Add the following line between <FileScanningTimeout> and <ExtendedConfig>: <SDDLogLevel>ERROR</SDDLogLevel>

  6. Save the changes to the XML configuration.

  7. Rerun the SDD operation to generate new Extractor logs with reduced logging levels.

  8. Monitor the new Extractor logs to ensure that they are no longer excessively large and that they contain fewer INFO level entries.
