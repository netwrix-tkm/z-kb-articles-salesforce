Summary:

  1. Nasuni error "Failed to enable auditing on Shares. Cannot access a disposed object."
  2. Entra ID. The search fails to display some Entra ID events.
  3. Not Recording Events for NFS Mount - Several Other servers are working new one is not.

Behavior Before Fix:

  1. A misleading error, "Failed to enable auditing on Shares. Cannot access a disposed object. Object name: System.Net.Http.HttpClient", occasionally appeared in the Console for Nasuni hosts. Despite the error, auditing was successfully enabled upon retry.
  2. The product collected all Entra ID activity correctly, but the search feature failed to display the following events:
    • Add member to role requested (PIM activation)
    • Add member to role completed (PIM activation)
    • Add member to role in PIM requested (timebound)
    • Add member to role in PIM completed (permanent)
    • Add eligible member to role in PIM completed (timebound)
  1. The product failed to report some Samba / SMB linux activity

Behavior After Fix:

  1. The Console suppresses transient errors during auditing configuration on Nasuni hosts. An error is only displayed if all attempts to configure auditing fail.
  2. The search feature displays all Entra ID events correctly.
  3. Samba / SMB linux activity is reported as expected.
