IMPORTANT: This is an unvalidated article. The product R&D team has yet to review some or all of information below.

Symptom

Netwrix Enterprise Auditor prompts the following error during the File Server Activity auditing (FSAC) scan: 

ERROR FILESYSTEMACCESS TFileSystemProcessor.RunFileSystemActivity
Error during processing: GetFSACIniAttributeValue: 
Error: Ini section for <HOST> does not have LOG_FILE \\\C$\Program Files\Stealthbits\StealthAUDIT\FSAC\SBTFileMon.ini

IMPORTANT: Depending on the Netwrix Activity Monitor version, the SBTFileMon.ini file can be located in one of the following folders:

For Activity Monitor v6.9 and earlier:

C:\Program Files\Stealthbits\StealthAUDIT\FSAC

For Activity Monitor v7.0 and newer:

C:\ProgramData\Netwrix\Activity Monitor\Agent

Cause

The SBTFileMon.ini file is corrupted.

Resolution

IMPORTANT: Keep copies of edited files to back up reference values.

Refer to the following steps to resolve the issue:

  1. Locate the affected SBTFileMon.ini file on the affected Activity Monitor agent.

  2. Replace the corrupted SBTFileMon.ini file with an .ini file from a working Activity Monitor agent.

  3. Modify the HOST= and LOG_FILE= values to match the intended values. Refer to the following example: 

    HOST=FILESERVER01
LOG_FILE=C:\ProgramData\Netwrix\Activity Monitor\Agent\ActivityLogs\FILESERVER01_Log_.tsv

  4. Verify that the replacement ini file is correctly configured.

  5. Test the Activity Monitor agent to ensure it can successfully collect FSAC scan data without encountering the error.

Alternatively, you can delete or rename the SBTFileMon.ini file to allow the Activity Monitor agent to recreate it. However, this approach would require reconfiguring the monitored host outputs for file and syslog and monitoring configurations, making it a slightly more time-consuming option.
