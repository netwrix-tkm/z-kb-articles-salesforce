Summary: FSAA Registry Key Error on NAS devices

Issue: "Error during processing: Could not open registry key SYSTEM\CurrentControlSet\Services\SBTLogging\Parameters for reading: Access is denied (5); local: False"

Instructions: If you get this error after running a FSAA scan this indicates that the FSAA scan was unable to open a registry key.  
 
Windows

If you get this after scanning a Windows host, either the registry key doesn't exist, or access is denied. To investigate

  • Open Regedit,
  • Start – CMD - Regedit (running as the same credential as the scan)
  • Try going to the location of the error message and see if the key exists or if you get access denied  

NAS

If you get this after scanning a NAS host, check the details in the Host Management

  • Open the Stealth Audit console
  • Find host management of the left hand side
  • All Hosts
  • Find the Host that has the issue
  • Select View\Edit Host (top right)
  • Select edit Host (bottom of the right)
  • Find OSName, OSType, Model and Manufacturer columns
  • Change OSName and OSType to 'NAS'.
  • Change Model and Manufacturer to 'N/A'.
