Issue: Issue has been identified when upgrading to SA 8.2 prior to bulk importing data that was scanned in previous versions of SA.
 
You may see an error on bulk import that says "Failed to import SA_FSAA_Resources: Column 'ID' does not allow DBNull.Value." or "Stealthbits.StealthAUDIT.DataCollectors.FSAA.FSAAException: ID 5 mapping not found in ResourceMap"
 
To fix this issue, a tool has been created to remedy the tier 2 databases that will allow the data to be imported.
 

Instructions: Drop in the contents of the zip file attached to this KB into the FSAA folder of your SA Console and follow the instructions in the ReadMe.

https://downloads.stealthbits.com/access/files/StealthTOOLS/AUDIT/ResetMinChangedIDsTool.zip

Once the tool is ran successfully, modify the USN value in the SA_FSAA_Hosts table in tier 1.
If running Permissions bulk import:
               UPDATE SA_FSAA_Hosts SET AccessUSN = 0 WHERE HOST = [HOST]
 
If running Activity bulk import:
               UPDATE SA_FSAA_Hosts SET ActivityUSN = 0 WHERE HOST = [HOST]
 
If running SDD bulk import:
               UPDATE SA_FSAA_Hosts SET DLPUSN = 0 WHERE HOST = [HOST]
