Summary:

  1. NAM - User SID resolves incorrectly
  2. NetApp. Occasional errors when an SVM is being monitored by several agents
  3. SUVP 25-02 support

Behavior Before Fix:

  1. Previously the search behavior was to resolve the account SID that started with the SID value. And this resolved incorrectly because there was another account starting with the same values.
  2. When two or more agents are monitoring the same SVM, they compete against each other to update the FPolicy configuration of the SVM. As a result, one of the agents may fail and display an error in the Console. The end result is that the FPolicy configuration is accurate and the error disappears in several minutes.
  3.  
  • 2025 Server - KB5051987
    • LDAP Bind events will be lost or incomplete
    • Logs: 
      • Couldn't resolve LDAP_CONN::BindRequest
      • Couldn't resolve CConnectionEx::InitializeClientData (4 param)|

  • 2022 Server - KB5051979
    •  NTLM, FSMO roles and LDAP Bind events will be lost or incomplete
    • Logs: 
      • Couldn't resolve NlpUserValidate

      • Couldn't resolve DsaGetValidFSMOs

      • Couldn't resolve LDAP_CONN::BindRequest

      • Couldn't resolve KdcGetTicket
      • Couldn't resolve HandleTGSRequest
      • Couldn't resolve I_GetASTicket
  • 2019 Server - KB5052000
    • NTLM, LDAP Bind and Kerberos events will be lost or incomplete
    • Logs: 
      • Couldn't resolve NlpUserValidate
      • Couldn't resolve LDAP_CONN::BindRequest
      • Couldn't resolve KdcGetTicket
      • Couldn't resolve HandleTGSRequest
      • Couldn't resolve I_GetASTicket
  • 2016 Server - KB5052006
    • NTLM and AD Replication events will be lost or incomplete
    • Logs: 
      • Couldn't resolve NlpUserValidate
      • Couldn't resolve IDL_DRSGetNCChanges

Behavior After Fix:

  1. We updated the search behavior to resolve the account SID that fully matches the SID value.
  2. Conflicts during FPolicy updates are not reported in the Console.
  3. AD Module updated. Product should report Active Directory events as expected
