Title:

activitymonitor-7.1.234

Summary:

  1. NAM - High CPU and RAM usage during scans
  2. Nasuni Connection Error post-upgrade to NAM v7.1.194
  3. SI - Domain Controller freeze every reboot

Behavior Before Fix:

  1. Customer reported that after upgrade from 6.0.1162 to 7.0.3099 CPU usage is at 60-70% and RAM usage is at 50%.
  2. Nasuni host would report a connection error related to SysinfoGet.
  3. Customer reported that domain controllers have frozen after reboots; A SI Agent was rebranded to the NTP product and default installation folder was changed to "c:\Program Files\Netwrix\". Due to it uses a shared file monitoring service with NAM and the NAM product still uses old 'Stealthbits' folder and cannot operate with the NTP agent together.

Behavior After Fix:

  1. Implemented many CPU and RAM optimizations for the agent and related services.
  2. Implemented fix related to restsharp and dependencies upgrades.
  3. For the compatibility with the NAM product the default folder for the file monitoring service was changed to old default "c: \Program Files\Stealthbits\".
