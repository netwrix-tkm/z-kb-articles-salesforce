Summary: This explains how StealthAUDIT selects credentials from Connection Profiles.

StealthAUDIT tries to match the domain in the Account column in the Connection Profile to:

  1. The Target Host's WindowsDomain, as visible in Host Management.
  2. The Target Host's DNSDomain, as visible in Host Management (though only if the Target Host's WindowsDomain value is blank). 

If neither match, each credential in the Connection Profile will be attempted in the order listed within the Connection Profile.
