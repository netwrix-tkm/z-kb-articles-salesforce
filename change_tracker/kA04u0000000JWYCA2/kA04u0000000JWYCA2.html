In this example, I will be using the FIM - File Integrity Monitoring Tracker. All paths listed as a tracked folder within a configuration template will be "baselined". 

Enabling the “Send baseline events” option on the FIM tracker (or any tracker which supports it) will effectively send back an event from the agent to the hub when a baseline scan is complete and it will allow you to see a list of the files and folders that have been captured as part of the baseline and the state of the file/folder i.e. its attributes. Whenever changes are then made to the files/folders, the baseline is updated and more baseline events will be sent to hub showing you the original baseline as “Historical” and the updated baseline as “Current”.
