2017-10-11 08:11:18,385 [4] ERROR ItemStoragePipeline - Error processing item, couldn't collect filehashsha256 for file: /etc/rc6.d/K05dsmcad Error: Failed to collect data for attribute (Could not find file "/etc/rc6.d/K05dsmcad")


What Does It Mean?

This error occurs when you have your configuration template in Change Tracker looking to track a file and the agent cannot hash the file because it is not found. In this case, we are looking to track "/etc/rc6.d/K05dsmcad".


How Can I Fix This / Should I Worry?

If you are recieving this for a file that actually does not exist, then no, do not worry. If you are using one template to track multiple agents, and some agents do have the file, while others dont, it is not a problem to let it track this file. The only caution to have with this is the amount of files it is looking for. If the template grows exponentially, having a large volume of these errors may cause performance issues, however, in most cases, it is perfectly fine.


If you are positive the file exists, but are getting this error, do the following

  1. Check your configuration template. Is it looking for the file in the same place you have placed the file?
  2. Is this a file that is in different places on different servers? if so, please define all of the potential spots in the template.
  3. Does this file have special permissions that do not allow it to be read by our agent? If so, adjust the file permissions, or adjust the privilages of the account our agent uses.


In most cases, you can simply ignore this error as it is not harmful.
