Installing NNT Agent App for Solaris (SPARC and Intel)

The NNT Change Tracker Enterprise Agent for Linux/Unix uses a Mono runtime instead of the Microsoft .NET framework used on Windows. NNT provide pre-built runtime file sets for Solaris hosts which can be simply downloaded and unzipped on the target host, along with the NNT AgentApp file set. Solaris will typically require the libstdc++ package to be added. This can be acquired from the official Oracle website or from {+}http://www.opencsw.org/+[ |http://www.opencsw.org/]

Step 1 – Downloads

Download the required files for your version of Solaris from the NNT Members Area[ |http://www.newnettechnologies.com/members-area/member-2/cat_view/6-support-information-for-existing-customers/23-nnt-agent-app-nix-runtime-packages.html]. This will almost certainly comprise of a GC and mono zip file. For example, Solaris 10 Intel requires the following:
gc-7.5-solaris-10-intel
Solaris 10 Intel Mono Runtime (Mono 3.2.3)
The NNT Agent installation file set (NNTAgent-1.0.0.xxx-yyy.zip can be downloaded from the Members Area. Copy the three zip files onto your host using scp / WinSCP. The file should be copied to the /opt directory.

Step 2 – Extract Mono runtime to host

Extract the mono runtime and GC zip files.
$ unzip gc-7.5-solaris-10-intel.zip
$ unzip mono-3.2.3-solaris-10-intel.zip
You should then have /opt/mono and /opt/gc directory in the root of /opt.

Step 4 – Extract NNT Agent App files to host

$ unzip NNTAgent-1.0.0.70.zip files/mono/ -d /opt/mono/nntagent*

Step 5 – Create HubDetails.xml file for the NNT Agent App

Note: entries in highlight should be edited to reflect your settings.
$ cat /opt/mono/nntagent/files/mono/HubDetails.xml


true
https://192.168.1.141/api/
agent
passWord121

EOF
Note: It is a good idea to add this HubDetails.xml to your NNT AgentApp file set so that it will then be included for any future deployments to other hosts.

Step 6 – Setup Path Information

NOTE: This should be achievable with one command as follows
$ export LD_LIBRARY_PATH=$LD_LIBRARY_PATH:/opt/mono/lib:/opt/gc/lib:/opt/csw/lib
The path '/opt/csw/lib' is present to provide the location of libstdc++.so.6 as per step 2.
Verify the path has been set correctly using
$ set | grep LD_LIB
LD_LIBRARY_PATH=:/opt/gc/lib:/opt/mono/lib:/opt/csw/lib

Step 7 – Run Agent

From the nntagent program files path (/opt/mono/nntagent/files/mono)
$ cd /opt/mono/nntagent/files/mono
$ nohup /opt/mono/bin/mono NNTAgentApp.exe &
