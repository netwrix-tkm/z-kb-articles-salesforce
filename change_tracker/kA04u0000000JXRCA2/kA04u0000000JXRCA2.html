If you have a system in your environment which represents the gold standard of required installed software, then you can use NNT Change Tracker to build a report which will check if other systems match the gold standard machine.

The approach in the below steps, isolates the gold standard machine within its own groups and with its own templates applied. This is so we can gather the information from the gold standard system without impacting the templates which are applied to the wider production environment.

Step 1 - Create a template

Identify the system to be used for the gold standard baseline report and confirm that this system is registered with Change Tracker. Use the Device tile to find the template that is applied to the system as we are going to make a copy of this template, CIS Win 8 L1 Base in this example.

While logged on with an admin account, move to Settings > Configuration Template.

Find the template which by default is applied to the system. In the example below I will be reviewing a Windows 8 system and so I have located the template, CIS Win 8 L1 Base.

Press the copy config button and provide the template with suitable name. I have called my template CIS Win 8 L1 Base Gold Standard.

 

Step 2 - Turn on baselining

Within the created gold standard template, we now need to configure the template to collect the baseline information that we will use to build our report. To do this edit your template and on the Installed Software and Updates tab, click the Send baseline events check box and the save changes button.

 

Step 3 - Create a Group Structure

Still logged in with the Admin account, move to Settings > Groups and create a group for your gold standard systems. In the example below I have created a group for a Window 8 system.

With the group created, move to the Device Templates tab and use the Add a Template to the Group button to import your gold standard template.

Step 4 - Add your device to the group

With both group and template created, move your device to into the group from Settings > Agents and Devices using the edit button.

 

Step 5 - Creating the baseline report

As soon as the agent is added into the group the agent will begin its baseline activities. You can see the Installed Software baseline event messages through the Change Tracker Event tab by using a Baseline event type within the filter.

 

Once the baselining has completed, which shouldn’t take more than a ten minutes, move to the Device tile and highlight the gold standard system. You will see a button under the configuration display entitled ‘Create Baseline Report’.

Click the create baseline report and accept the default from the popup windows, obviously change the name of the report if you prefer!

 

You have now created a baseline report for all software and patches that are installed on your gold standard system. You can see the report from Settings > Compliance Reports Templates.

Step 6 - Running the report

With the report created you will be able to add the report to an existing group, in order to run the report against all occupants of that group, or run the report from ad hoc from the Device tile.
