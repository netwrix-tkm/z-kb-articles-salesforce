Below is a description of the files contained within the agent installation path as of agent version 7.0.0.32:

  • Agent.id‚Äì When an agent is installed, it is assigned a unique ID and this is where it is contained.
  • MasterDevice.json ‚ÄìThis contains the details of the master proxy device which is always the local agent. It also contains the parameters on how the agent interacts with remote systems/databases that it has been tasked with monitoring.
  • ProxiedDevices.json ‚Äì This file contains an array of all the devices being proxied by the agent. For devices where you cannot install the agent, they will be proxied so for that reason the agent stores the names of the devices currently being proxied by the agent for its record.
  • Events.db - This stores the events data captured by Change Tracker.
  • Config.xml ‚Äì This file contains the devices configuration data. It holds information of what directories, registry items and security policies are being tracked by the agent.
  • Store.db ‚Äì All of the baseline information is stored here.
  • Rolling-log.txt ‚Äì This file contains log data that has been generated by the agent that reports to Change Tracker. It is helpful for troubleshooting when issues arise as it holds data such as any errors that have occurred with the agent, when compliance reports run, if there are any error with the compliance report that have run, when the agent goes online/offline, etc.
  • HubDetails.xml‚Äì This file contains details related to the Change Tracker server and this information allows the agent to be able to authenticate to the Change Tracker server. Information such as the username, password, HUB URL, proxy information, etc. are stored in here.
