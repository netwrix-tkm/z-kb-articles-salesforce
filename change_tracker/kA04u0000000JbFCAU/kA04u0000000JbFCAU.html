This document is to provide assist the end user in providing us with as much information as possible about problems related to Agents or ITSM Module.

General Information to Provide

This is some of the general information NNT would like to know regardless of what the problem is.


  1. A detailed description of the issue that is being experienced. If a screenshot/video can be captured, this will also be useful, as well as when the issue was noticed.

  2. What version of the NNT Hub Server are you using?
  3. Have there been any recent updates, or patches to the operating system?
    1. If so, what was updated?
    2. How recently did it occur?
    3. Can information about what was installed be provided?
    4. Was the system rebooted after the patch?

  4. Information about the scope of the problem
    1. Is the issue related to a specific group of devices hosting the same application? Are all agents of this group impacted or is it only a few of them?
    2. Is the issue related to a specific group of devices hosting the same operating system? Are all agents of this group impacted or is it only a few of them?
    3. How many agents are impacted?

  5. If all agents are offline, where any of the following network settings changed?
    1. SSL/TLS Version
    2. Mocrosoft IIS Services
    3. IIS Certificate
    4. IIS Website Port Binding
    5. MongoDB Version
    6. .NET Framework / Mono Runtime Version

Linux Gen7Agent / ExpressAgent Information and Attachments

If the issue is related to a Linux Agent, here is the information that needs to be provided in addition to the general information.

  1. Compress and Attach "/var/messages" file to the ticket
  2. Compress and Attach "/var/nnt/" folder to the ticket
  3. Compress and Attach "/opt/nnt" folder to the ticket

  4. Output of "uname -a"

    Example Output:

    [root@NNT-LINUX-CENTOS7-D3 ~]$ uname -a
