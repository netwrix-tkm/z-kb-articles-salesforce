Example: You are monitoring syslog messages being generated by a Windows Server. One of the messages that you have specifically correlated is for when administrator accounts fail to login to the system. Within each of these events, there is an account name listed. Seeing multiple messages which contain the same account name within a short time period, may indicate a brute force attack. You would like to setup an alert which will generate a ticket but only if they see the same account name within a message, 5 times within a 30 second period. In order to do this, please follow the instructions below:

Log Tracker Version 5-8-1 is required in order for you to use this functionality.

  • Log into Log Tracker
  • Select the Alerts tab
  • Select the Custom tab
  • Select the Wizard > button
  • Set Custom Alert Title to:
  • Select the Next > button
  • Set the Custom Alert Type to: Parse Value Alert
  • Select the Next > button
  • Set your Correlation Thread to: “an account failed to log on” events>
  • Set the Parse Expression to: Account Name: *
  • Select the Next > button
  • Set the Compare Function to: (GE) Greater Than Or Equal
  • Set the Threshold to: 5
  • Set the Execute Interval to: 30
  • Select the Next > button
  • Set the Alert Facility to:
  • Set the Alert Severity to:
  • Set Assign Ticket To User to:
  • Set Send Clear Severity to:
  • Select the Next > button
  • Set the Alert Message/Ticket Text to:
  • Select the Finish! button
