This function is only available from the localhost and therefore can only be run by a high privilege user with logon rights to the Change Tracker host server

CentOS Based Hub

From a command line interface on the CentOS host itself, enter the following command:

$ wget --no-check-certificate -qO- https://localhost/api/resetadminpassword

This will write the reset password to a file called resetpassword or resetpassword.x if multiple resets have been attempted. The file will be in the directory from which the wget command was run.

Windows Based Hub

Using the browser on the Windows Hub machine itself enter the following into the address bar: 

https://localhost/api/resetadminpassword



The password displayed will be your new password. At next logon to the Change Tracker Gen 7 web GUI you will be able to logon using the reset password but will also be prompted to change the password to one of your choice.If you are not prompted to change your password, follow the instructions below.

If you where not prompted to change your password

Sometimes you will not be prompted to change your password. In order to manually change it, you will need to log in using your new passsword, and proceed by going to the "Settings Tile"

Once on the "Settings" page. Click on "Users" on the left navigation panel. This should bring up the User Administration page on your screen. To reset the password, click on the blue paperclip next to the account name you want to reset (In this case, it would be "admin")



Upon clicking the blue paperclip, you will be prompted to type a new password, and confirm this password. Once you have done this, click "Apply" - Your new password should be immediately available for use.
