Overview

Sometimes, companies need to create a dedicated user with some level of admin access to perform MongoDB administrative activities like backup and support in cases of issues. This article explains how to create a user of such type.

Instructions

Follow the steps below to create a user with admin level access:

  • Please ensure that during the creation of the user that you specify a unique and secure username and password.

    1. RDP to the server hosting Change Tracker.

    2. Browse to the folder: C:\Program Files\NNT Change Tracker Suite\Gen7\MongoDB\bin.

    3. Open a Command Prompt window.

    4. Open up mongo shell executing the following command: mongo.exe

    5. Enter the commands below:

      use admin

      db.createUser(

      {

      user: "admin",

      pwd: "password",

      roles: [ { role: "root", db: "admin" } ]

      }

      );

  • If an admin user has been created on the Mongo database previously and the password for that user needs to be changed, follow the steps below:

    The ‘user’ and ‘new_password’ values will need to be set to match the customers environment.

    1. RDP to the server hosting Change Tracker.

    2. Browse to the folder: C:\Program Files\NNT Change Tracker Suite\Gen7\MongoDB\bin

    3. Open a Command Prompt window.

    4. Open up mongo shell executing the following command: mongo.exe

      IMPORTANT: If authentication is enabled already, include these details as part of the command: mongo.exe -u admin -p password --authenticationDatabase admin

    5. Enter the commands below:

      use admin

      db.changeUserPassword(‘user’,’new_password’)
